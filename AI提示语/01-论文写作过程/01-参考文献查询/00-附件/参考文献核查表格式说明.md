## 四、DOI错误处理流程

### 4.1 DOI格式错误

当遇到DOI格式错误时，按以下步骤处理：

```markdown
### Step 1: 验证DOI格式
- [ ] 检查前缀：10.开头
- [ ] 检查斜杠数量：仅一个
- [ ] 检查后缀长度：足够长

### Step 2: 格式修复
- [ ] 去除 "https://doi.org/" 前缀
- [ ] 去除 "doi:" 前缀
- [ ] 统一为 "10.xxxx/xxxxx" 格式

### Step 3: 验证修复后的DOI
- [ ] 在 Crossref 验证
- [ ] 在 Semantic Scholar 验证
- [ ] 记录原始DOI → 修复后DOI
```

### 4.2 DOI失效处理

当DOI指向的资源不可用时：

```markdown
### 情况A: 论文换用新DOI
- [ ] 在 Crossref 搜索标题
- [ ] 找到最新DOI
- [ ] 记录: 原DOI → 新DOI

### 情况B: 论文撤稿或撤回
- [ ] 标记⚠️部分验证
- [ ] 联系原作者确认
- [ ] 考虑替代论文

### 情况C: DOI指向网页而非论文
- [ ] 提取网页中的论文信息
- [ ] 使用论文实际DOI
- [ ] 在正文中更新引用
```

### 4.3 无DOI论文处理

当论文没有DOI时：

```markdown
- [ ] 使用 arXiv ID（如果有）
- [ ] 使用 PubMed ID（生物医学领域）
- [ ] 使用完整的期刊/会议信息替代
- [ ] 在"问题类型"标记"无DOI"
```

## 五、状态标识说明

### 5.1 核查状态

| 状态 | 含义 | 符号 |
|:---|:---|:---:|
| 已验证 | 文献信息准确，可直接使用 | ✅ |
| 部分验证 | 存在疑问，需进一步核实 | ⚠️ |
| 已替代 | 原文献无法确认，已提供替代方案 | ✅ |
| 未通过 | 核查未通过，无法使用 | ❌ |

### 5.2 问题类型

| 问题类型 | 说明 |
|:---|:---|
| DOI确认 | DOI真实存在，信息准确 |
| 信息准确 | 通过多数据库验证，信息准确 |
| 原文献无法确认 | 作者/标题/年份无法与数据库记录匹配 |
| 需进一步核实 | 存在匹配疑问，需联系原作者或进一步查证 |
| 引用格式错误 | 格式不符合GB/T 7714要求 |
| 无DOI | 论文无DOI，需使用其他标识 |
| DOI失效 | DOI指向资源不可用 |

## 六、PDF文件命名规则

### 6.1 命名格式

```
[编号]_[论文标识]_[发表年份].pdf
```

### 6.2 组成部分

| 部分 | 说明 | 示例 |
|:---|:---|:---|
| `[编号]` | 原文编号，替代用"[原编号]替代" | `[1]`, `[2替代]` |
| `_` | 下划线分隔符 | `_` |
| `[论文标识]` | 论文简短标识（作者姓氏或算法名） | `DeepSeek`, `PPO`, `TD3` |
| `_` | 下划线分隔符 | `_` |
| `[发表年份]` | 4位数字年份 | `2024`, `2017` |
| `.pdf` | 文件扩展名 | `.pdf` |

### 6.3 命名示例

| 场景 | 文件名 | 说明 |
|:---|:---|:---|
| 期刊论文 | `[4]_AttitudeControl_Drones_2024.pdf` | 无人机姿态控制论文 |
| 会议论文 | `[6]_DeepDrone_RSS_2020.pdf` | 无人机特技表演 |
| 替代论文 | `[2替代]_UAVSurvey_IEEE_2019.pdf` | 替代的UAV综述 |
| arXiv预印本 | `[1]_DeepSeek_GRPO_arXiv_2024.pdf` | 含GRPO的DeepSeek-V2 |

### 6.4 命名原则

1. **唯一性**：每个文件有唯一命名
2. **可识别性**：命名反映论文核心内容
3. **有序性**：按编号排序
4. **兼容性**：仅使用字母、数字、下划线

## 七、核查流程checklist

```markdown
## 核查流程

### Step 1: 初筛
- [ ] 记录原文引用信息（作者、标题、期刊/会议、年份、DOI）
- [ ] 检查引用格式完整性
- [ ] 检查DOI格式是否正确

### Step 2: 数据库验证
- [ ] 在arXiv/OpenAlex确认DOI/标题
- [ ] 在Crossref验证元数据
- [ ] 在Semantic Scholar核对作者信息
- [ ] 记录验证数据库列表

### Step 3: 问题判定
- [ ] 如信息准确 → 标记✅已验证
- [ ] 如存在疑问 → 标记⚠️部分验证，列出具体问题
- [ ] 如无法确认 → 标记❌，启动替代论文搜索
- [ ] 如DOI失效 → 按4.2节流程处理

### Step 4: PDF获取
- [ ] 检查Unpaywall开放获取链接
- [ ] 下载PDF至指定目录
- [ ] 按命名规则重命名文件

### Step 5: 修正建议
- [ ] 如需修正，生成国标格式引用
- [ ] 提供替代论文（如适用）
- [ ] 更新正文中的引用位置

### Step 6: 可信度评估
- [ ] 评估DOI完整性
- [ ] 评估数据库匹配数
- [ ] 评估PDF来源
- [ ] 综合评分
```

## 八、国标引用格式（GB/T 7714-2015）

### 8.1 期刊文章

```
[编号] 作者. 题目[J]. 期刊名, 年份, 卷(期): 页码.
```

**示例：**
```
[1] DeepSeek-AI, Liu A, Feng B, et al. DeepSeek-V2: A Strong, Economical, and Efficient Mixture-of-Experts Language Model[J]. arXiv:2405.04434, 2024.
```

### 8.2 会议论文

```
[编号] 作者. 题目[C]//会议名称. 年份: 页码.
```

**示例：**
```
[3] Clarke S G, Hwang I. Deep Reinforcement Learning Control for Aerobatic Maneuvering of Agile Fixed-Wing Aircraft[C]//AIAA Scitech 2020 Forum, 2020: 1-12.
```

### 8.3 arXiv预印本

```
[编号] 作者. 题目[OL]. (年份-MM-DD). arXiv:XXXXX. URL
```

**示例：**
```
[5] Schulman J, Wolski F, Dhariwal P, et al. Proximal Policy Optimization Algorithms[OL]. (2017-07-20). arXiv:1707.06347. https://arxiv.org/abs/1707.06347
```

### 8.4 专著

```
[编号] 作者. 书名[M]. 出版社, 年份.
```

### 8.5 学位论文

```
[编号] 作者. 题目[D]. 学位授予单位, 年份.
```

### 8.6 专利

```
[编号] 专利权人. 题目[P]. 专利号, 日期.
```

## 十、替代论文判定规则

当原文献无法确认时，替代论文应满足以下标准：

### 10.1 领域相关性（必须）

- 与原文献研究领域一致
- 与原文献主题高度相关
- 优先选择同一研究方向

### 10.2 发表时间（参考）

- 优先选择5年内发表的论文
- 经典工作可放宽至10年
- 记录发表年份及选择理由

### 10.3 质量指标（参考）

| 指标 | 要求 |
|:---|:---|
| 引用次数 | 优先高引用论文 |
| 期刊/会议级别 | 优先核心期刊、顶级会议 |
| 开放获取 | 优先OA论文 |

### 10.4 替代记录格式

```markdown
**替代原因：** [原文献无法确认的具体原因]

**替代论文：**
| 字段 | 值 |
|:---|:---|
| 作者 | [作者] |
| 题目 | [题目] |
| 期刊/会议 | [期刊/会议] |
| 年份 | [年份] |
| DOI | [DOI] |

**替代理由：** [为什么选择此论文作为替代]
```

## 十一、可信度评分标准

### 11.1 评分维度

| 维度 | 说明 | 权重 |
|:---|:---|:---:|
| DOI完整性 | DOI是否存在且有效 | 25% |
| 数据库匹配数 | 多少个数据库确认该文献 | 25% |
| PDF来源 | 是否有官方PDF或OA来源 | 25% |
| 引用格式规范 | 是否符合GB/T 7714格式 | 25% |

### 11.2 评分等级

| 等级 | 综合得分 | 含义 |
|:---:|:---:|:---|
| ★★★★★ | 90-100 | 极高可信度，可直接使用 |
| ★★★★☆ | 70-89 | 高可信度，轻微问题可修正 |
| ★★★☆☆ | 50-69 | 中可信度，需确认后使用 |
| ★★☆☆☆ | 30-49 | 低可信度，建议替代 |
| ★☆☆☆☆ | 0-29 | 极低可信度，不建议使用 |


## 任务目标

根据实际硬件连接方式，将工程从 LCD1602 并口显示改为 7 针 SPI OLED 显示，并把 GPS 串口从 USART1 改到 PA2/PA3 对应的 USART2。

## 思考过程

用户给出的硬件是 7 针 SPI OLED：PB8=SCK、PB9=SDA、PB12=DC、PB13=CS，OLED RES 接小系统板 R 复位脚。当前固件却使用 LCD1602 并口 PB10-PB15，并使用 USART1，因此屏幕不亮的主要原因是驱动和接线完全不匹配。选择使用 GPIO 模拟 SPI 驱动常见 SSD1306 OLED，避免引入 SPI 外设配置和 CubeMX 重生成。

## 遇到的问题

当前代码包含 `lcd1602.c/h`，并在 `main.c` 中调用 LCD1602 API。GPS 串口当前为 USART1，但用户接线为 PA2/PA3，即 USART2。OLED RES 接系统复位脚，软件无法单独控制 OLED reset，只能在初始化前等待复位完成。

## 解决方法

新增 SPI OLED 驱动，使用 GPIOB8/GPIOB9/GPIOB12/GPIOB13 模拟 SSD1306 4 线 SPI。替换 `main.c` 中的 LCD1602 显示调用，并将 UART 初始化、IRQ 和回调逻辑改为 USART2。

具体按如下理解适配接线：

PB8 -> OLED D0/SCK

PB9 -> OLED D1/SDA

PB12 -> OLED DC

PB13 -> OLED CS

OLED RES -> 小系统板 R/RESET

PA2/USART2_TX -> GPS RX

PA3/USART2_RX -> GPS TX

## 验证结果

Debug 构建通过：

`RAM: 2264 B / 20 KB`

烧录成功，STM32CubeProgrammer 输出 `Download verified successfully`，并执行 `MCU Reset` / `Software reset is performed`。

新固件启动后 OLED 应显示：

第一行：`GPS WAITING`

第二行：`USART2 9600`

## 代码变更摘要

新增本记录文件，用于记录本次硬件接线适配过程。

新增 `Core/Inc/oled_spi.h` 和 `Core/Src/oled_spi.c`：

实现基于 GPIO 模拟 SPI 的 SSD1306 OLED 初始化、清屏和文本行显示。

修改 `Core/Src/main.c`：

显示接口从 LCD1602 改为 OLED；串口从 USART1 改为 USART2；GPS 接收回调改为检查 USART2。

修改 `Core/Src/stm32f1xx_hal_msp.c`：

USART 初始化从 PA9/PA10 的 USART1 改为 PA2/PA3 的 USART2，并启用 USART2 中断。

修改 `Core/Src/stm32f1xx_it.c`：

中断处理函数从 `USART1_IRQHandler` 改为 `USART2_IRQHandler`。

修改 `cmake/stm32cubemx/CMakeLists.txt`：

从构建中移除 `lcd1602.c`，加入 `oled_spi.c`。

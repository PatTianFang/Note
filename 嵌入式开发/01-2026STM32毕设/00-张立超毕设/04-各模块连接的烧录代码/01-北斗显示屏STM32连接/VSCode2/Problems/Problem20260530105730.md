## 任务目标

将当前 STM32 工程的 Debug 固件烧录到已连接的设备中。

## 思考过程

优先使用当前工程已有的 CMake Debug 产物，并通过本机已安装的 STM32CubeProgrammer CLI 走 ST-Link/SWD 烧录。烧录前先确认设备连接和固件产物，再重新构建，避免写入过期固件。

## 遇到的问题

初始检查时需要确认是否有 ST-Link 连接、是否存在可烧录固件，以及应使用哪个构建配置。检测到一个 ST-Link，序列号为 `37FF71064E5734368D191B43`。当前可用固件为 `build/Debug/VSCode.elf`。

目标芯片通过 SWD 正常识别，电压为 `3.27V`，设备 ID 为 `0x410`，设备名为 `STM32F101/F102/F103 Medium-density`。

## 解决方法

使用 ST 已安装的 `STM32_Programmer_CLI.exe` 检测连接设备，并准备使用 `port=SWD` 连接目标。烧录前执行 Debug preset 构建。

执行 Debug 构建后，Ninja 返回 `no work to do`，说明当前构建产物已是最新状态。随后通过 ST-Link/SWD 写入 `build/Debug/VSCode.elf`，并启用写入后校验和复位。

## 验证结果

烧录成功。STM32CubeProgrammer 输出固件大小为 `12.62 KB`，写入地址为 `0x08000000`。

校验成功，输出 `Download verified successfully`。

复位成功，输出 `MCU Reset` 和 `Software reset is performed`。

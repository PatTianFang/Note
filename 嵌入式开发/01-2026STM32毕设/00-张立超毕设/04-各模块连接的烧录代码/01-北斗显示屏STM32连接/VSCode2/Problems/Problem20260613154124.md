## 任务目标

为当前 VS Code 工作区配置 ST 官方 STM32 插件使用的 STM32CubeMX 可执行文件路径，使插件能够从工程中打开 `.ioc` 文件或调用 CubeMX。

## 思考过程

假设当前用户希望配置的是本工作区，而不是全局 VS Code 用户设置。这样改动范围最小，不会影响其它 STM32 项目。

已确认 `D:\STM32CubeMX\STM32CubeMX.exe` 文件存在。通过检查已安装的 ST 官方 VS Code 插件 `package.json`，确认 STM32CubeMX 路径设置键为 `stm32cube-ide-core.configuration.productSTM32CubeMX.executablePath`。

## 遇到的问题

第一次使用 `rg` 查询插件配置时，PowerShell 下直接传入带通配符的扩展目录路径导致路径语法错误。

## 解决方法

改为先枚举 `C:\Users\PatTi\.vscode\extensions` 下的 `stmicroelectronics*` 扩展目录，再逐个查询 `package.json`，最终定位到正确配置键。

## 验证结果

`.vscode/settings.json` 已能被 PowerShell `ConvertFrom-Json` 正常解析。

读取 `stm32cube-ide-core.configuration.productSTM32CubeMX.executablePath` 得到：

```text
D:\STM32CubeMX\STM32CubeMX.exe
```

并通过 `Test-Path` 确认可执行文件存在。

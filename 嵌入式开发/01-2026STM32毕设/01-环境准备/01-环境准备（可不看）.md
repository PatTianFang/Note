# 01-环境准备

> 适合对象：第一次接触 STM32、还不知道要安装哪些软件的人。  
> 更新时间：2026-05-30。软件下载请以官网页面显示的最新版本为准，不建议从来历不明的网盘或论坛安装包开始。

## 1. 先看结论：新手应该怎么装

如果你现在只是想把 STM32 开发环境搭起来，先按下面这一套来，不要一开始就把所有工具都装满。

推荐路线：

```text
STM32CubeMX 配置芯片和引脚
        ↓
STM32CubeIDE 写代码、编译、调试
        ↓
STM32CubeProgrammer 烧录、擦除、检查芯片连接
        ↓
串口工具 查看打印信息、调试 GPS/无线/通信模块
```

最少需要准备的软件：

| 软件 | 是否必装 | 用来做什么 | 官方入口 |
| --- | --- | --- | --- |
| STM32CubeIDE | 必装，推荐新手使用 | 写代码、编译、下载、单步调试 | [ST 官方 STM32CubeIDE](https://www.st.com/en/development-tools/stm32cubeide.html) |
| STM32CubeMX | 必装，建议单独安装 | 图形化配置芯片、引脚、时钟、外设，生成初始化代码 | [ST 官方 STM32CubeMX](https://www.st.com/en/development-tools/stm32cubemx.html) |
| STM32CubeProgrammer | 必装 | 单独烧录程序、擦除芯片、升级 ST-LINK、排查下载失败 | [ST 官方 STM32CubeProgrammer](https://www.st.com/en/development-tools/stm32cubeprog.html) |
| ST-LINK USB Driver | 视情况安装 | 让 Windows 识别 ST-LINK 调试器 | [ST 官方 STSW-LINK009](https://www.st.com/en/development-tools/stsw-link009.html) |
| STM32Cube MCU Package | 必装对应芯片系列 | HAL/LL 库、启动文件、例程、芯片支持包 | [STM32CubeF1 示例入口](https://www.st.com/en/embedded-software/stm32cubef1.html) |

| Visual Studio Code | 你已经熟悉 VS Code，或后期想用轻量编辑器 | 写代码、看代码、配合 ST 插件开发 | [VS Code 官方下载](https://code.visualstudio.com/Download) |

## 2. 新手先理解几个词

STM32 开发不是只装一个“写代码软件”就结束，它通常由几类工具配合完成。

| 名词 | 大白话解释 | 你什么时候会遇到 |
| --- | --- | --- |
| IDE | 写代码的软件，类似“开发工作台” | 打开工程、写 C 代码、点 Build、点 Debug |
| 编译器 | 把 C 代码翻译成单片机能运行的机器码 | 点击 Build 或 Compile 时 |
| 调试器 | 让电脑和 STM32 芯片通信，可以单步运行、看变量 | 点击 Debug 时 |
| 烧录器 | 把编译好的程序写进芯片 Flash | 点击 Download、Program、Run 时 |
| ST-LINK | ST 常用调试/烧录硬件 | Nucleo 板自带，很多开发板需要外接 |
| SWD | STM32 常用调试接口，只需要少量线 | 接 ST-LINK 时常见 SWDIO、SWCLK、GND |
| UART/USART | 串口通信 | 打印日志、GPS 模块、蓝牙/WiFi/无线模块常用 |
| HAL 库 | ST 提供的外设驱动库，适合入门 | CubeMX 生成代码后经常看到 `HAL_` 开头函数 |
| LL 库 | 更底层、更接近寄存器的 ST 库 | 后期需要更高效率时可能用 |
| CMSIS | ARM Cortex-M 的基础支持层 | 启动文件、内核头文件、异常中断相关 |
| `.ioc` 文件 | CubeMX 的工程配置文件 | 双击可以重新配置引脚、时钟、外设 |
| `.elf` 文件 | 带调试信息的编译结果 | IDE 调试时常用 |
| `.hex` / `.bin` 文件 | 可烧录到芯片的程序文件 | CubeProgrammer 或量产烧录时常用 |

## 3. 电脑和目录准备

### 3.1 电脑要求

建议电脑满足：

- Windows 10 或 Windows 11，64 位系统。
- 至少 8 GB 内存，16 GB 更舒服。
- 至少预留 20 GB 磁盘空间。CubeIDE、CubeMX、芯片库、例程和工程会逐渐占空间。
- 有管理员权限。安装驱动、ST-LINK、USB 转串口驱动时通常需要管理员权限。
- 有稳定网络。ST 的安装包和芯片库比较大，有些下载需要注册 ST 账号。

### 3.2 建议创建固定目录

不要把工程随手放在桌面、微信文件夹、QQ 文件夹、OneDrive 同步目录里。建议新建一个专门的 STM32 工作目录，例如：

```text
D:\STM32
D:\STM32\Projects
D:\STM32\Repository
D:\STM32\Tools
D:\STM32\Datasheets
D:\STM32\Installers
```

各目录用途：

| 目录 | 用途 |
| --- | --- |
| `D:\STM32\Projects` | 放你自己写的工程 |
| `D:\STM32\Repository` | 放 STM32Cube MCU Package 芯片库 |
| `D:\STM32\Tools` | 放临时工具或绿色软件 |
| `D:\STM32\Datasheets` | 放芯片手册、模块资料、原理图 |
| `D:\STM32\Installers` | 保存安装包，方便重装和记录版本 |

注意：

- 工程路径尽量使用英文、数字、下划线，不要有中文、空格、特殊符号。
- 安装路径可以用软件默认路径，不要随意改到很深的目录。
- 工程目录不要放到云同步目录里，避免编译中间文件被同步软件锁住。

推荐工程路径示例：

```text
D:\STM32\Projects\F103_Beep
D:\STM32\Projects\F103_GPS
D:\STM32\Projects\Graduation_Project
```

不推荐：

```text
C:\Users\你的名字\Desktop\新建文件夹\毕设\最终版\最终版2\代码
D:\微信文件\老师发的\单片机代码
D:\OneDrive\STM32工程
```

## 4. 必装软件详细说明

### 4.1 STM32CubeIDE：写代码、编译、调试

用途：

- 新建 STM32 工程。
- 写 C 代码。
- 调用编译器生成可执行程序。
- 通过 ST-LINK 下载程序到芯片。
- 单步调试，查看变量、寄存器、内存。

为什么新手推荐它：

- 它是 ST 官方工具，和 STM32CubeMX、STM32CubeProgrammer 配合最顺。
- 不需要一开始单独折腾编译器、链接脚本、调试配置。
- 新手可以先用图形界面完成大部分操作。

下载：

- 官方页面：[STM32CubeIDE](https://www.st.com/en/development-tools/stm32cubeide.html)
- 下载时可能需要登录或注册 myST 账号。

安装建议：

- Windows 版选择 64 位安装包。
- 安装路径使用默认值即可。
- 第一次启动时选择工作区，例如：

```text
D:\STM32\Workspace
```

使用建议：

- 第一次用它时，不要急着改复杂设置。
- 看到 `Debug`、`Release` 两种构建配置时，入门阶段先用 `Debug`。
- 不要删除自动生成的启动文件、链接脚本、`Core`、`Drivers` 目录。

### 4.2 STM32CubeMX：配置芯片、引脚、时钟、外设

用途：

- 选择 STM32 具体型号，例如 `STM32F103C8Tx`。
- 配置引脚功能，例如 GPIO、USART、I2C、SPI、ADC、PWM。
- 配置系统时钟，例如外部晶振 HSE、PLL 倍频。
- 配置中断、DMA、外设参数。
- 生成初始化代码。

为什么重要：

STM32 外设多、寄存器多，新手直接手写初始化代码很容易出错。CubeMX 可以用图形界面先把“芯片怎么接线、外设怎么开”配置出来，再生成一个能跑起来的工程骨架。

下载：

- 官方页面：[STM32CubeMX](https://www.st.com/en/development-tools/stm32cubemx.html)

安装后必须做的一件事：

进入 CubeMX 的软件包管理界面，安装你芯片对应的 STM32Cube MCU Package。

如果你使用的是常见的 `STM32F103C8T6`：

- 芯片系列是 `STM32F1`。
- 需要安装 `STM32CubeF1`。
- 官方入口：[STM32CubeF1](https://www.st.com/en/embedded-software/stm32cubef1.html)

建议把 Repository 路径设置为：

```text
D:\STM32\Repository
```

这样芯片库不会散落在系统默认目录里。

### 4.3 STM32CubeProgrammer：烧录、擦除、排查连接

用途：

- 把 `.hex`、`.bin`、`.elf` 文件烧录进 STM32。
- 擦除芯片。
- 查看芯片 Flash、Option Bytes。
- 通过 ST-LINK、UART Bootloader、USB DFU 等方式连接芯片。
- 升级 ST-LINK 固件。

为什么新手也要装：

即使你平时在 CubeIDE 里点 Debug 下载程序，遇到“下载失败”“芯片连不上”“芯片被读保护锁住”时，CubeProgrammer 是最直接的排查工具。

下载：

- 官方页面：[STM32CubeProgrammer](https://www.st.com/en/development-tools/stm32cubeprog.html)

常见使用方式：

- 连接方式选择 `ST-LINK`。
- Port 选择 `SWD`。
- 频率先用较低值，例如 4000 kHz、1000 kHz，连接不稳定时再降。
- 连接成功后能看到芯片型号、Flash 大小、Device ID。

### 4.4 ST-LINK USB Driver：让 Windows 识别调试器

用途：

- 让电脑识别 ST-LINK 调试器。
- 解决设备管理器里出现未知设备、驱动异常的问题。

下载：

- 官方页面：[STSW-LINK009](https://www.st.com/en/development-tools/stsw-link009.html)

什么时候需要单独安装：

- 插上开发板或 ST-LINK 后，设备管理器没有出现 ST-LINK。
- CubeIDE 或 CubeProgrammer 提示找不到 ST-LINK。
- Windows 把调试器识别成未知 USB 设备。

说明：

- 有些新版 ST 工具会顺带安装驱动，但遇到识别失败时仍建议单独装一次官方驱动。
- 如果你使用的是盗版或兼容 ST-LINK，驱动和固件升级可能表现不稳定。毕业设计阶段建议尽量使用可靠的 ST-LINK 或带正版 ST-LINK 的 Nucleo/Discovery 板。

### 4.5 STM32Cube MCU Package：芯片支持包

用途：

- 提供 HAL/LL 驱动库。
- 提供 CMSIS 头文件和启动文件。
- 提供链接脚本、外设例程、板级例程。
- 让 CubeMX 知道这个系列芯片有哪些引脚、外设和限制。

怎么选：

| 芯片型号开头 | 对应软件包 |
| --- | --- |
| STM32F0xx | STM32CubeF0 |
| STM32F1xx | STM32CubeF1 |
| STM32F4xx | STM32CubeF4 |
| STM32G0xx | STM32CubeG0 |
| STM32G4xx | STM32CubeG4 |
| STM32H7xx | STM32CubeH7 |
| STM32L4xx | STM32CubeL4 |
| STM32U5xx | STM32CubeU5 |

本项目资料中出现了 `STM32F103C8T6`，所以至少要准备：

```text
STM32CubeF1
```

## 5. 强烈建议安装的软件

### 5.1 串口工具：查看输出、调试模块

STM32 开发中，串口是最常用的调试手段之一。你可能会用它做这些事：

- 让 STM32 打印 `printf` 日志。
- 查看 GPS/北斗模块输出的 NMEA 数据。
- 给蓝牙、WiFi、无线数传模块发送 AT 指令。
- 验证两个串口设备之间能不能通信。

推荐工具：

| 工具 | 特点 | 官方入口 |
| --- | --- | --- |
| Tera Term | 轻量，适合串口入门 | [Tera Term](https://teratermproject.github.io/) |
| MobaXterm | 功能多，也能 SSH、串口 | [MobaXterm](https://mobaxterm.mobatek.net/download.html) |

串口基础参数：

| 参数 | 常见值 | 说明 |
| --- | --- | --- |
| Baud Rate | `9600`、`115200` | 波特率，双方必须一致 |
| Data Bits | `8` | 数据位 |
| Parity | `None` | 校验位 |
| Stop Bits | `1` | 停止位 |
| Flow Control | `None` | 入门阶段通常关闭 |

常见接线：

```text
USB 转串口 TX  -> STM32 RX
USB 转串口 RX  -> STM32 TX
USB 转串口 GND -> STM32 GND
```

注意：

- TX 和 RX 要交叉接。
- GND 必须共地。
- 不要把 5V 串口信号直接接到不耐 5V 的 STM32 引脚。
- 如果模块标的是 `RS232`，不要直接接 STM32 的 TTL 串口，需要 MAX3232 之类的电平转换芯片。

### 5.2 USB 转串口驱动

很多 USB 转 TTL 模块插上电脑后会显示成 COM 口。常见芯片和驱动如下：

| 芯片 | 常见模块名 | 官方驱动 |
| --- | --- | --- |
| CH340 / CH341 | 便宜 USB 转 TTL 模块常用 | [WCH CH341SER](https://www.wch.cn/downloads/CH341SER_EXE.html) |
| CP2102 / CP2104 | 稳定性较好，很多模块使用 | [Silicon Labs CP210x](https://www.silabs.com/developer-tools/usb-to-uart-bridge-vcp-drivers) |
| FT232 | FTDI USB 转串口芯片 | [FTDI VCP](https://ftdichip.com/drivers/vcp-drivers/) |

怎么判断是否安装成功：

1. 插入 USB 转串口模块。
2. 打开 Windows 设备管理器。
3. 查看“端口 COM 和 LPT”。
4. 能看到 `USB-SERIAL CH340`、`Silicon Labs CP210x`、`USB Serial Port` 等类似名称。
5. 记下 COM 号，例如 `COM5`，串口工具里要选择这个端口。

### 5.3 Git：保存代码历史

用途：

- 每完成一个功能就保存一次版本。
- 改坏代码后可以回到之前能运行的状态。
- 写毕业设计时方便记录开发过程。
- 后期可以配合 GitHub、Gitee 备份。

下载：

- 官方页面：[Git for Windows](https://git-scm.com/download/win)

新手安装建议：

- 大多数选项保持默认。
- 默认编辑器可以选 VS Code，如果你安装了 VS Code。
- 换行符选项不懂就保持推荐默认值。

安装后检查：

打开 PowerShell 输入：

```powershell
git --version
```

能显示版本号即可。

最基本使用习惯：

```powershell
git init
git status
git add .
git commit -m "init stm32 project"
```

说明：如果你还没学 Git，可以先不用命令行，但一定要养成“每个能运行的版本都备份”的习惯。

## 6. 可选软件怎么选

### 6.1 Keil MDK：老师或老教程常用

Keil MDK 是很多 STM32 老教程使用的 IDE。如果你的课程、老师、例程明确要求 Keil，就安装它。

用途：

- 打开 `.uvprojx` 工程。
- 编译和调试 ARM Cortex-M 单片机项目。
- 配合 ST-LINK 或 J-Link 下载程序。

下载：

- 官方入口：[Keil/Arm](https://www.keil.arm.com/)

什么时候不建议一开始装：

- 你没有明确的 Keil 工程需求。
- 你使用的是 CubeIDE 教程。
- 你还没弄清楚芯片型号、编译器、Pack、调试器之间的关系。

如果必须用 Keil：

- 安装 MDK 后还要安装对应芯片的 Device Family Pack。
- CubeMX 生成工程时，`Toolchain / IDE` 选择 `MDK-ARM`。
- 如果例程很老，注意它可能使用标准外设库 SPL，而不是 HAL 库。

### 6.2 VS Code：更像编辑器，不是完整替代品

VS Code 本身只是编辑器，不是完整的 STM32 开发环境。它需要配合插件、编译器、调试器、CMake 或 ST 的命令行工具。

适合：

- 你已经熟悉 VS Code。
- 你想写代码体验更轻量。
- 项目后期需要 Git、Markdown、脚本、CMake 一起管理。

不适合：

- 你完全第一次接触 STM32。
- 你还不知道编译器、调试器、烧录器分别是什么。

下载：

- 官方页面：[Visual Studio Code](https://code.visualstudio.com/Download)

建议安装的扩展：

- C/C++。
- Cortex-Debug。
- CMake Tools。
- ST 官方 STM32 相关扩展，按官方页面和扩展市场说明安装。

### 6.3 STM32CubeCLT、CMake、Ninja、Arm GNU Toolchain

这些是命令行和现代构建环境会用到的工具。

入门阶段：

- 如果你使用 CubeIDE，可以先不装。
- 如果你使用 VS Code + CMake，就需要认真配置。

作用：

| 工具 | 作用 |
| --- | --- |
| STM32CubeCLT | ST 官方命令行工具链集合，适合配合 VS Code |
| CMake | 生成构建系统，不直接编译代码 |
| Ninja | 被 CMake 调用的快速构建器 |
| Arm GNU Toolchain | 提供 `arm-none-eabi-gcc`、`gdb` 等命令行工具 |

检查命令：

```powershell
cmake --version
ninja --version
arm-none-eabi-gcc --version
```

注意：如果你只安装了 CubeIDE，`arm-none-eabi-gcc` 不一定能直接在 PowerShell 里运行，因为它可能没有加入系统 PATH。这不代表 CubeIDE 不能编译。

## 7. 推荐安装顺序

按这个顺序装，出问题最容易定位。

### 第 1 步：创建目录

创建：

```text
D:\STM32
D:\STM32\Projects
D:\STM32\Repository
D:\STM32\Installers
D:\STM32\Datasheets
```

### 第 2 步：注册或准备 ST 账号

ST 的部分软件下载需要登录。建议提前准备一个邮箱注册 myST 账号。

注意：

- 尽量使用官网链接。
- 下载慢时可以换浏览器或网络，不要随便下载不明来源的重打包安装器。

### 第 3 步：安装 STM32CubeIDE

安装后打开一次，确认能进入主界面。

建议工作区：

```text
D:\STM32\Workspace
```

### 第 4 步：安装 STM32CubeMX

安装后设置 Repository 路径：

```text
D:\STM32\Repository
```

然后安装芯片包。若使用 `STM32F103C8T6`：

```text
STM32CubeF1
```

### 第 5 步：安装 STM32CubeProgrammer

安装后打开，先不用急着烧录。等 ST-LINK 和开发板连接好后，用它确认能连上芯片。

### 第 6 步：安装或确认 ST-LINK 驱动

插入 ST-LINK 或开发板后，在设备管理器确认有没有 ST-LINK 设备。

如果没有，再安装：

```text
STSW-LINK009
```

### 第 7 步：安装串口工具和 USB 转串口驱动

如果你手上有 USB 转 TTL 模块，看芯片型号：

- CH340/CH341：装 WCH 驱动。
- CP2102：装 Silicon Labs 驱动。
- FT232：装 FTDI 驱动。

插上模块后确认 COM 口出现。

### 第 8 步：安装 Git 和解压工具

安装 Git for Windows。

安装 7-Zip 或 WinRAR，用于解压 `.rar`、`.zip` 资料包。

### 第 9 步：按需要安装 Keil 或 VS Code

只有在下面情况才需要：

- 教程明确要求 Keil。
- 老师给的是 Keil 工程。
- 你想用 VS Code 管理代码。
- 你要做 CMake 或命令行构建。

## 8. 第一次验证：确认环境真的可用

软件装完不代表环境已经能开发。建议按下面顺序检查。

### 8.1 检查 ST-LINK 是否识别

操作：

1. 插入开发板或 ST-LINK。
2. 打开设备管理器。
3. 查看是否有 ST-LINK 相关设备。
4. 打开 STM32CubeProgrammer。
5. 连接方式选择 `ST-LINK`。
6. Port 选择 `SWD`。
7. 点击 `Connect`。

成功现象：

- CubeProgrammer 能连上。
- 能看到芯片信息。
- 能读取 Flash 或 Option Bytes。

失败时先检查：

- USB 线是不是数据线，不是只能充电的线。
- ST-LINK 的 `GND` 是否和目标板 `GND` 相连。
- `SWDIO`、`SWCLK` 是否接反。
- 目标板是否供电。
- BOOT0 是否处于正常启动位置。
- SWD 频率是否太高，可以降到 1000 kHz 或更低。

### 8.2 检查 STM32CubeMX 能否生成工程

以 `STM32F103C8T6` 为例：

1. 打开 STM32CubeMX。
2. 选择 `Access to MCU Selector`。
3. 搜索 `STM32F103C8Tx`。
4. 创建工程。
5. 在 `SYS` 中把 Debug 设置为 `Serial Wire`。
6. 配置一个 GPIO 输出，例如常见小系统板 LED 可能接在 `PC13`。
7. 在 `Project Manager` 中选择工具链为 `STM32CubeIDE`。
8. 生成代码。

成功现象：

- 生成的工程目录中有 `.ioc` 文件。
- 有 `Core`、`Drivers` 等目录。
- 可以导入或打开到 CubeIDE。

### 8.3 检查 CubeIDE 能否编译

操作：

1. 打开 CubeIDE。
2. 导入或打开刚才生成的工程。
3. 点击 Build。

成功现象：

- 控制台没有 Error。
- 生成 `Debug` 目录。
- 能看到 `.elf`、`.hex` 或 `.bin` 文件。

### 8.4 检查能否下载和运行

操作：

1. 连接 ST-LINK 和开发板。
2. 在 CubeIDE 中点击 Debug。
3. 如果进入调试界面，点击 Resume/Run。

可以用最简单的 LED 闪烁确认程序运行：

```c
while (1)
{
    HAL_GPIO_TogglePin(GPIOC, GPIO_PIN_13);
    HAL_Delay(500);
}
```

注意：

- 不同开发板 LED 引脚不一定是 `PC13`，请看原理图或开发板资料。
- 如果使用外部晶振 HSE，频率要和板子实际晶振一致。常见 F103 小系统板可能是 8 MHz，但必须以你的板子为准。

### 8.5 检查串口工具能否打开 COM 口

操作：

1. 插入 USB 转串口模块。
2. 打开设备管理器，记下 COM 号。
3. 打开 Tera Term 或 MobaXterm。
4. 选择对应 COM 口。
5. 设置波特率，例如 `115200`。

成功现象：

- 串口工具能打开端口，不提示占用。
- 如果 STM32 程序有串口打印，能看到输出。

## 9. 本项目可能特别需要注意的地方

从已有资料看，项目可能会涉及：

- `STM32F103C8T6` 小系统板。
- 北斗/GPS 模块。
- 无线通信或语音/数传模块。

因此环境准备时重点关注：

1. 芯片包至少安装 `STM32CubeF1`。
2. 串口工具必须准备好，因为 GPS、无线模块经常用 UART 通信。
3. USB 转 TTL 模块和驱动要准备好，方便单独测试模块。
4. 如果模块接口写的是 `RS232`，不要直接接 STM32，需要电平转换。
5. 多模块联调前，先单独验证每个模块能和电脑串口工具通信。

## 10. 常见问题和处理方法

| 问题 | 可能原因 | 处理方法 |
| --- | --- | --- |
| CubeProgrammer 找不到 ST-LINK | 驱动没装、USB 线不是数据线、ST-LINK 损坏 | 换数据线，安装 STSW-LINK009，换 USB 口，检查设备管理器 |
| 提示 `No target connected` | SWD 接线错误、目标板没供电、SWD 频率过高 | 检查 GND、SWDIO、SWCLK、3.3V，降低 SWD 频率 |
| 下载后程序不运行 | BOOT0 错误、时钟配置错误、芯片型号选错 | BOOT0 拉低，核对晶振频率和芯片型号 |
| CubeMX 找不到芯片包 | 没安装 MCU Package、网络问题 | 在 CubeMX 包管理器安装对应系列，也可以从 ST 官网下载 |
| `.ioc` 打不开 | CubeMX 没装或版本太旧 | 安装或升级 STM32CubeMX |
| 工程路径报错 | 路径有中文、空格或太深 | 移到 `D:\STM32\Projects\ProjectName` |
| 串口乱码 | 波特率不一致、GND 未共地、接线反了 | 核对波特率、TX/RX 交叉、GND 共地 |
| 串口打不开 | COM 口被占用 | 关闭其他串口工具、CubeIDE 串口监视器、调试助手 |
| Keil 编译报缺少设备包 | 没装 Device Family Pack | 在 Keil Pack Installer 安装对应芯片包 |
| 程序能烧录但不能调试 | Debug 口被代码关闭、低功耗配置影响 | CubeMX 中 `SYS -> Debug` 选择 `Serial Wire` |

## 12. 初学者建议的学习顺序

环境准备好后，不要直接做完整毕业设计。建议按下面顺序练习：

1. 点亮 LED。
2. 按键输入。
3. 串口发送 `hello stm32`。
4. 串口接收数据。
5. 定时器中断。
6. PWM 输出。
7. I2C 或 SPI 驱动一个简单模块。
8. 读取 GPS/北斗模块串口数据。
9. 调试无线通信模块。
10. 最后再把多个模块组合成毕业设计系统。

每完成一步，都保存一个能运行的版本。

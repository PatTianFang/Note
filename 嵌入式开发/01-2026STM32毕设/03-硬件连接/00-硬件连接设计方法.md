# 00-硬件连接设计方法

> 更新时间：2026-05-30。  
> 本文不针对某一个具体项目，而是总结做 STM32 硬件连接时通用的设计方法、思考顺序、资料查阅路径和检查清单。  
> 核心原则：不要先接线，先设计；不要凭感觉，先查资料；不要一次全接，先分模块验证。

## 1. 先建立正确的硬件连接思维

硬件连接不是“看见几个引脚就拿杜邦线接上”。

正确思路是：

```text
需求是什么
        ↓
需要哪些模块
        ↓
每个模块需要什么电源
        ↓
每个模块用什么通信协议
        ↓
这些协议需要哪些 STM32 引脚
        ↓
这些引脚是否冲突
        ↓
电平是否匹配
        ↓
画连接表
        ↓
单模块验证
        ↓
整机联调
```

硬件设计最重要的 5 句话：

1. 先看电源，再看信号。
2. 先看电平，再接 IO。
3. 先看协议，再选引脚。
4. 先单独验证，再整体联调。
5. 所有接线必须和软件配置一致。

## 2. 设计前先问自己 10 个问题

每接一个模块前，先回答下面问题。

| 问题 | 为什么要问 |
| --- | --- |
| 这个模块是干什么的？ | 明确它在系统里的角色 |
| 它需要几伏供电？ | 防止烧模块或烧 MCU |
| 最大电流是多少？ | 判断电源能不能带得动 |
| 它的 IO 电平是多少？ | 判断能不能直接接 STM32 |
| 它用什么通信方式？ | UART/I2C/SPI/ADC/PWM/GPIO 接法完全不同 |
| 每个引脚方向是什么？ | TX/RX、MOSI/MISO、输入/输出不能接反 |
| 是否需要上拉/下拉？ | I2C、按键、BOOT、RESET 常见 |
| 是否需要复位脚/使能脚？ | 有些模块不拉 EN/RST 就不工作 |
| 是否占用调试引脚？ | 避免影响 ST-LINK 下载调试 |
| 有没有初始化时序要求？ | 有些屏幕、传感器、通信模块需要延时和复位 |

如果这些问题答不上来，不要急着接线，先查资料。

## 3. 应该查阅哪些资料

硬件连接设计至少要查 6 类资料。

### 3.1 模块数据手册或用户手册

这是最重要的资料。

你要从里面找：

- 引脚定义。
- 供电电压。
- IO 电平。
- 典型电流和峰值电流。
- 通信协议。
- 默认波特率、I2C 地址、SPI 模式。
- 是否需要上拉电阻。
- 是否需要复位、使能、启动配置。
- 初始化命令或时序。
- 典型应用电路。

重点看这些章节：

```text
Pin Definition
Electrical Characteristics
Recommended Operating Conditions
Interface Description
Timing Diagram
Application Circuit
```

### 3.2 STM32 芯片 Datasheet

Datasheet 主要看芯片外部特性。

你要查：

- 引脚复用功能。
- 每个引脚能不能做 UART/I2C/SPI/ADC/PWM。
- 哪些引脚是 5V tolerant。
- IO 最大电流。
- 供电范围。
- 封装引脚编号。
- Flash/RAM 容量。
- 时钟引脚。
- BOOT、NRST、SWD 调试引脚。

Datasheet 解决的问题是：

```text
这个 STM32 引脚从硬件上能不能接这个模块？
```

### 3.3 STM32 Reference Manual

Reference Manual 主要看内部外设怎么工作。

你要查：

- UART、I2C、SPI、ADC、TIM 等外设功能。
- 外设挂在哪条总线。
- 寄存器说明。
- 时钟树。
- 中断。
- DMA。
- Boot 模式。

Reference Manual 解决的问题是：

```text
这个外设在 STM32 内部应该怎么配置？
```

### 3.4 开发板原理图

如果你用的是小系统板或开发板，一定要看原理图。

你要查：

- STM32 哪些引脚已经被板载 LED、按键、晶振占用。
- 哪些引脚被引出到排针。
- 电源从哪里来。
- 3.3V 稳压芯片能提供多大电流。
- BOOT0 怎么接。
- NRST 怎么接。
- SWD 接口怎么接。
- 是否有 USB 转串口。
- 板载晶振频率是多少。

开发板原理图解决的问题是：

```text
这个引脚在我这块板子上到底能不能用？
```

### 3.5 协议文档

如果模块输出的是标准协议，需要查协议文档。

例子：

| 模块 | 常见协议 |
| --- | --- |
| GPS/北斗 | NMEA |
| 温湿度传感器 | I2C、自定义单总线 |
| OLED | SSD1306/SH1106 指令集 |
| 蓝牙/WiFi | AT 指令 |
| RS485 模块 | Modbus RTU |
| CAN 模块 | CAN 帧格式 |

协议文档解决的问题是：

```text
收到的数据是什么意思，应该怎么解析？
```

### 3.6 示例代码和官方例程

例程不是让你直接无脑复制，而是用来确认：

- 官方推荐怎么接线。
- 初始化顺序是什么。
- 默认通信参数是什么。
- 有哪些坑。

看例程时重点看：

- 用了哪个外设。
- 用了哪些引脚。
- 波特率/I2C 地址/SPI 模式。
- 是否打开中断。
- 是否用了 DMA。
- 初始化函数调用顺序。

## 4. 硬件连接设计的标准流程

### 4.1 第 1 步：画系统框图

先不要画复杂原理图，先画框图。

例如：

```text
电源模块
   ↓
STM32 主控
   ├── 串口模块
   ├── 显示模块
   ├── 传感器模块
   ├── 通信模块
   └── 下载调试接口
```

框图要回答：

- 谁是主控。
- 谁给谁供电。
- 谁和谁通信。
- 通信方向是什么。
- 哪些模块必须一直工作。
- 哪些模块只是调试时使用。

### 4.2 第 2 步：列模块清单

建立模块表：

| 模块 | 功能 | 供电 | 通信方式 | 备注 |
| --- | --- | --- | --- | --- |
| MCU | 主控 | 3.3V | 多种外设 | STM32 |
| 显示屏 | 显示信息 | 3.3V/5V | I2C/SPI | 看具体屏幕 |
| 传感器 | 采集数据 | 3.3V | I2C/SPI/ADC | 看模块手册 |
| 通信模块 | 对外通信 | 3.3V/5V | UART/SPI | 关注电流和电平 |

这个表能帮你快速发现：

- 需要几个串口。
- 需要几个 I2C。
- 需要几个 SPI。
- 电源是否够。
- 引脚是否够。

### 4.3 第 3 步：确定电源树

电源树要先于信号设计。

你需要确定：

- 系统输入电源是多少，USB、锂电池、适配器还是其他。
- STM32 用 3.3V 还是其他电压。
- 外设模块是否都能用 3.3V。
- 有没有 5V 模块。
- 3.3V 稳压芯片最大输出电流是多少。
- 所有模块峰值电流加起来是否超限。

简单电源预算表：

| 模块 | 电压 | 典型电流 | 峰值电流 | 备注 |
| --- | --- | --- | --- | --- |
| STM32 | 3.3V |  |  | 看工作频率和外设 |
| 显示屏 | 3.3V |  |  | OLED 亮度影响电流 |
| 通信模块 | 3.3V/5V |  |  | 峰值可能很高 |
| 传感器 | 3.3V |  |  | 多数较小 |

设计原则：

```text
电源能力 > 所有模块峰值电流总和，并留余量。
```

### 4.4 第 4 步：检查电平兼容

STM32 常见 IO 是 3.3V 逻辑。

检查内容：

- 模块输出高电平是不是 STM32 能接受。
- STM32 输出高电平模块能不能识别。
- 模块是否输出 5V 信号。
- STM32 该引脚是否 5V 容忍。
- 模拟信号是否超过 ADC 量程。

常见危险：

```text
5V 模块 TX 直接接 STM32 RX。
5V 传感器模拟输出直接接 STM32 ADC。
RS232 电平直接接 STM32 UART。
```

如果电平不兼容，需要：

- 电平转换芯片。
- 分压电阻。
- 光耦隔离。
- MAX3232 之类的 RS232/TTL 转换。
- RS485 收发器。

### 4.5 第 5 步：确定通信协议

不同协议设计重点不同。

| 协议/接口 | 特点 | 设计重点 |
| --- | --- | --- |
| UART | TX/RX 串行通信 | 波特率、TX/RX 交叉、共地 |
| I2C | SCL/SDA 两线总线 | 上拉电阻、地址冲突、总线电容 |
| SPI | SCK/MOSI/MISO/CS | 模式、片选、线序、速度 |
| GPIO | 普通输入输出 | 上拉/下拉、默认状态、驱动能力 |
| ADC | 模拟输入 | 电压范围、滤波、采样阻抗 |
| PWM | 定时器输出 | 频率、占空比、引脚复用 |
| USB | 差分高速信号 | 阻抗、ESD、时钟精度 |
| CAN/RS485 | 工业通信 | 收发器、终端电阻、方向控制 |

先定协议，再选 STM32 引脚。

### 4.6 第 6 步：分配 STM32 引脚

引脚分配要同时考虑：

- 芯片引脚是否支持该外设功能。
- 开发板是否把这个引脚引出来。
- 是否和板载电路冲突。
- 是否和 ST-LINK 调试引脚冲突。
- 是否和外部晶振、复位、BOOT 冲突。
- 是否后期还要扩展。

建议使用 CubeMX 做引脚冲突检查。

不要轻易占用：

```text
PA13 / SWDIO
PA14 / SWCLK
NRST
BOOT0
OSC_IN / OSC_OUT
PC14 / PC15，若使用 LSE
```

除非你明确知道后果。

### 4.7 第 7 步：写连接表

正式接线前必须写连接表。

模板：

| 序号 | 功能 | STM32 引脚 | 模块引脚 | 方向 | 电压 | 协议 | 备注 |
| --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | 电源 | 3.3V | VCC | 供电 | 3.3V | 电源 | 确认电流 |
| 2 | 地 | GND | GND | 共地 | 0V | 电源 | 必接 |
| 3 | 数据 | PAx | TX/RX/SDA | 看方向 | 3.3V | UART/I2C/SPI | 不要接反 |

连接表是硬件和软件之间的契约。

后续 CubeMX、代码、接线都必须和这张表一致。

### 4.8 第 8 步：画原理图或接线图

哪怕只是杜邦线实验，也建议画图。

可以画：

- 手绘草图。
- draw.io。
- KiCad 原理图。
- Fritzing。
- 表格加文字说明。

图里必须能看出：

- VCC。
- GND。
- 每根信号线。
- 接口方向。
- 模块名称。
- 引脚编号。

不要只写“接到某个模块”，要写清楚接到模块哪个引脚。

### 4.9 第 9 步：软件配置同步

硬件接线确定后，要同步到软件。

需要检查：

- CubeMX 中引脚功能是否和接线一致。
- UART 波特率是否和模块一致。
- I2C 地址是否正确。
- SPI 模式是否正确。
- GPIO 输出默认电平是否安全。
- 中断是否打开。
- DMA 是否需要。
- 时钟是否满足外设要求。

硬件接线和软件配置不一致时，程序通常不会报错，但硬件就是不工作。

### 4.10 第 10 步：分阶段验证

不要一次性把所有模块接上。

推荐顺序：

```text
电源检查
        ↓
STM32 最小系统
        ↓
下载调试接口
        ↓
单个外设模块
        ↓
两个模块组合
        ↓
完整系统联调
```

每一步都要有明确成功标准。

例如：

- 电源检查：3.3V 正常，无短路。
- 下载调试：ST-LINK 能连接。
- UART：能收到稳定文本。
- I2C：能扫描到设备地址。
- SPI：屏幕能显示固定图案。
- ADC：读数随电压变化。

## 5. 各类接口的设计要点

### 5.1 UART

UART 常见于 GPS、蓝牙、WiFi、串口屏、调试日志。

需要确认：

- 波特率。
- 数据位。
- 校验位。
- 停止位。
- 是否需要流控。
- 电平是 TTL、RS232 还是 RS485。

接线规则：

```text
模块 TX -> STM32 RX
模块 RX -> STM32 TX
模块 GND -> STM32 GND
```

常见错误：

- TX/RX 没交叉。
- 波特率不一致。
- 没有共地。
- RS232 直接接 STM32。
- 模块输出 5V TTL。

### 5.2 I2C

I2C 常见于传感器、OLED、EEPROM。

需要确认：

- SCL/SDA 引脚。
- I2C 地址。
- 总线速度，100 kHz 或 400 kHz。
- 是否已有上拉电阻。
- 是否有地址冲突。

I2C 需要上拉电阻：

```text
SCL -> 上拉到 3.3V
SDA -> 上拉到 3.3V
```

常见错误：

- 没有上拉。
- 上拉到 5V。
- 地址写错。
- 多个模块地址相同。
- 线太长导致波形差。

### 5.3 SPI

SPI 常见于 OLED、Flash、传感器、无线模块。

需要确认：

- SCK。
- MOSI。
- MISO。
- CS。
- SPI 模式，CPOL/CPHA。
- 最高时钟频率。
- 多设备时每个设备要独立 CS。

常见错误：

- MOSI/MISO 接反。
- CS 没拉对。
- 模式不匹配。
- 速度太高。
- 模块实际是 I2C 屏，却按 SPI 接。

### 5.4 GPIO

GPIO 看似简单，但也容易出错。

需要确认：

- 输入还是输出。
- 上拉还是下拉。
- 默认电平。
- 推挽还是开漏。
- 是否需要消抖。
- 是否能驱动负载。

注意：

```text
STM32 IO 不能直接驱动大电流负载。
电机、继电器、蜂鸣器、大功率 LED 通常要加三极管/MOSFET/驱动芯片。
```

### 5.5 ADC

ADC 用于读取模拟电压。

需要确认：

- 输入电压范围不能超过 VDDA。
- 信号源阻抗不能太大。
- 是否需要滤波。
- 是否需要分压。
- 采样时间是否足够。
- 模拟地是否干净。

常见错误：

- 5V 模拟信号直接进 3.3V ADC。
- 没有共地。
- 分压阻值太大导致采样不准。
- 数字噪声影响模拟读数。

### 5.6 PWM

PWM 常用于舵机、蜂鸣器、背光、电机控制。

需要确认：

- 频率。
- 占空比范围。
- 定时器通道。
- 输出引脚复用。
- 负载是否需要驱动器。

特别注意：

```text
PWM 引脚输出的是信号，不是电源。
不能直接带大电机。
```

## 6. 电源和地的设计思想

### 6.1 电源先行

所有硬件问题里，电源问题最常见，也最隐蔽。

设计时先问：

- 输入电源从哪里来。
- 需要哪些电压。
- 3.3V 谁来产生。
- 5V 是否需要。
- 稳压芯片电流够不够。
- 模块峰值电流是多少。
- 是否需要大电容。
- 是否需要隔离或保护。

### 6.2 去耦电容

模块电源附近通常需要电容。

常见做法：

- 每个芯片 VCC 附近放 `0.1uF` 去耦电容。
- 模块电源入口放 `10uF` 或更大的电容。
- 电流波动大的模块适当增加电容。

作用：

- 抑制电源瞬间波动。
- 降低噪声。
- 防止模块启动时电压下跌。

### 6.3 地线

所有模块必须共地。

但是共地不等于乱接地。

要注意：

- 大电流负载的地线不要和敏感模拟信号共用细长路径。
- 通信线旁边最好有可靠地参考。
- 面包板和长杜邦线会引入不稳定因素。

### 6.4 上电顺序

有些模块对上电顺序敏感。

需要查：

- 是否有 EN 引脚。
- 是否有 RESET 引脚。
- 上电后需要延时多久。
- MCU 初始化前模块是否会输出异常电平。

如果模块有复位脚，最好让 MCU 能控制它，方便软件重启模块。

## 7. 引脚选择的原则

### 7.1 优先使用硬件外设引脚

例如：

- 串口模块优先接硬件 UART。
- I2C 设备优先接硬件 I2C。
- 高速 SPI 设备优先接硬件 SPI。
- PWM 输出优先接定时器通道。

软件模拟可以用，但不应该一开始滥用。

### 7.2 避免关键系统引脚

慎用：

```text
PA13 / PA14：SWD 调试
NRST：复位
BOOT0：启动模式
OSC_IN / OSC_OUT：外部晶振
PC14 / PC15：低速晶振
USB D+ / D-：USB
```

占用这些引脚可能导致：

- 不能下载。
- 不能调试。
- 不能启动。
- 时钟异常。
- USB 不工作。

### 7.3 预留扩展能力

不要把所有好用的引脚一次用完。

设计时预留：

- 一个串口用于调试日志。
- 一些普通 GPIO。
- 一个 I2C 或 SPI 扩展接口。
- 下载调试接口。

毕业设计后期经常会临时加功能，预留接口能减少返工。

## 8. 连接表和命名规范

硬件连接设计必须能被别人看懂，也能被未来的自己看懂。

推荐命名：

| 名称 | 含义 |
| --- | --- |
| `GPS_TX` | GPS 模块发送到 MCU |
| `GPS_RX` | MCU 发送到 GPS 模块 |
| `OLED_SCL` / `OLED_SCK` | OLED 时钟 |
| `OLED_SDA` / `OLED_MOSI` | OLED 数据 |
| `OLED_CS` | OLED 片选 |
| `OLED_DC` | OLED 命令/数据 |
| `OLED_RST` | OLED 复位 |
| `DBG_TX` | 调试串口发送 |
| `KEY1` | 按键 |
| `LED_RUN` | 运行指示灯 |

注意方向命名：

```text
GPS_TX 表示 GPS 模块的 TX。
它应该接 MCU 的 RX。
```

不要写模糊名字：

```text
线1
线2
数据
模块脚
```

## 9. 上电前检查方法

上电前一定检查。

### 9.1 目视检查

检查：

- VCC 有没有接错。
- GND 有没有漏接。
- TX/RX 有没有接反。
- 插针有没有错位一格。
- 杜邦线有没有松。
- 模块方向有没有插反。

### 9.2 万用表检查

断电状态：

- 测 VCC 和 GND 是否短路。
- 测关键电源线是否连通。
- 测地线是否连通。

上电状态：

- 测 3.3V 是否正常。
- 测 5V 是否正常。
- 测模块 VCC 是否真的有电。
- 测稳压芯片是否发烫。

### 9.3 限流上电

如果有可调电源，建议限流。

例如：

```text
3.3V，限流 100mA 或按系统需求设置
```

如果一上电电流异常升高，立即断电。

## 10. 分阶段调试方法

### 10.1 最小系统先跑通

只接：

- STM32。
- 电源。
- ST-LINK。

目标：

- 能下载程序。
- 能运行 LED 闪烁。
- 能复位。

### 10.2 单模块验证

一次只接一个模块。

例如：

- 只接 OLED，显示固定文本。
- 只接 GPS，用串口看 NMEA。
- 只接传感器，读取 ID。
- 只接通信模块，发送 AT 指令。

单模块不通，不要进入整机联调。

### 10.3 两两组合

组合测试：

- MCU + 显示。
- MCU + 传感器。
- MCU + 通信模块。
- MCU + 电源模块。

这样能定位问题来自哪一部分。

### 10.4 整机联调

最后再接所有模块。

整机联调时重点看：

- 电源是否下降。
- 串口是否丢数据。
- I2C 是否卡死。
- SPI 显示是否异常。
- 中断是否冲突。
- CPU 是否被某个任务阻塞。

## 12. 常见硬件错误

| 错误 | 后果 |
| --- | --- |
| 不共地 | 通信不稳定或完全不通 |
| TX/RX 没交叉 | UART 收不到数据 |
| 5V 信号直接进 STM32 | 可能烧 IO |
| I2C 没上拉 | 总线不工作 |
| SPI CS 没控制好 | 设备不响应 |
| 占用 SWD 引脚 | ST-LINK 不能下载调试 |
| 外部晶振配置和硬件不一致 | 程序可能不运行 |
| 模块电流超过稳压能力 | 复位、死机、数据异常 |
| 一次接太多模块 | 出问题无法定位 |
| 不看模块手册 | 接错电源或协议 |

## 13. 设计文档应该留下什么

硬件连接设计不是接完就结束。你需要留下文档，方便复现和排错。

至少保留：

1. 系统框图。
2. 模块清单。
3. 电源预算表。
4. 引脚分配表。
5. 接线总表。
6. CubeMX 配置说明。
7. 上电检查记录。
8. 分模块验证记录。
9. 常见问题和解决方法。
10. 版本变更记录。

推荐文档结构：

```text
01-系统目标
02-模块清单
03-电源设计
04-接口协议
05-引脚分配
06-接线总表
07-CubeMX配置
08-上电检查
09-分步验证
10-问题记录
```

## 15. 最终思想总结

硬件连接设计的核心不是“会不会接线”，而是“能不能证明这样接是对的”。

判断一个硬件连接设计是否合格，看它是否满足：

- 有资料依据。
- 电源正确。
- 电平兼容。
- 协议匹配。
- 引脚无冲突。
- 软件配置一致。
- 能单模块验证。
- 有问题能定位。
- 有文档能复现。

最后记住这条顺序：

```text
查资料 -> 定电源 -> 定电平 -> 定协议 -> 选引脚 -> 画表 -> 接线 -> 单测 -> 联调 -> 记录
```

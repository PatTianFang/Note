# 01-总体思路：STM32 + 北斗定位 + OLED 显示屏硬件连接怎么设计

> 更新时间：2026-05-30。  
> 本文以当前项目已有资料为基础，示例目标是：`STM32F103C8T6 小系统板 + 北斗/GPS 模块 + 7 针 SPI OLED`。  
> 重点不是只给一张接线表，而是讲清楚硬件连接设计时应该先想什么、怎么选接口、怎么验证、哪里容易接错。

## 1. 先明确系统要完成什么功能

目标功能：

```text
北斗/GPS 模块接收卫星信号
        ↓
模块通过串口输出 NMEA 定位数据
        ↓
STM32 通过 UART 接收数据
        ↓
STM32 解析 RMC 定位语句
        ↓
STM32 把时间、经纬度、定位状态显示到 OLED
```

整个系统可以分成 4 个部分：

| 模块 | 作用 |
| --- | --- |
| STM32F103C8T6 | 主控，负责接收、解析、显示控制 |
| 北斗/GPS 模块 | 定位，输出 NMEA 数据 |
| OLED 显示屏 | 显示定位状态、时间、经纬度 |
| ST-LINK | 下载程序、调试程序 |

核心思想：

```text
先确定每个模块需要什么电源、用什么通信协议、占用哪些 STM32 引脚。
然后画连接表。
最后按“电源 -> 单模块 -> 两模块 -> 整机”的顺序验证。
```

## 2. 总体连接框图

```text
                 北斗/GPS 模块
              ┌────────────────┐
 天线 ───────> │ 卫星定位接收     │
              │ UART TX/RX      │
              └───────┬────────┘
                      │ USART2
                      │ PA3 接收 NMEA
                      │ PA2 可发送配置命令
                      │
┌─────────────────────▼─────────────────────┐
│              STM32F103C8T6 小系统板        │
│                                             │
│  USART2：接收北斗/GPS NMEA 数据             │
│  GPIO 模拟 SPI：驱动 OLED                   │
│  SWD：连接 ST-LINK 下载和调试               │
└──────────────┬──────────────────────┬──────┘
               │                      │
               │ GPIO 模拟 SPI         │ SWD
               │                      │
        ┌──────▼──────┐        ┌──────▼──────┐
        │ 7 针 SPI OLED│        │   ST-LINK   │
        │ 显示定位信息 │        │ 下载/调试    │
        └─────────────┘        └─────────────┘
```

## 3. 设计硬件连接时先看 5 件事

不要一上来就接线。每接一个模块前，先确认下面 5 件事。

| 要确认的事 | 为什么重要 |
| --- | --- |
| 电源电压 | 3.3V 模块不能随便接 5V |
| 信号电平 | STM32F103 是 3.3V 逻辑，外部模块信号也要匹配 |
| 通信协议 | UART、I2C、SPI、普通 GPIO 接法完全不同 |
| 引脚方向 | TX/RX 要交叉，SPI 的 SCK/SDA/DC/CS 不能接错 |
| 是否和调试接口冲突 | PA13/PA14 是 SWD，尽量不要占用 |

建议你每次新增模块都先写一张表：

| 模块引脚 | 作用 | 电压 | 接 STM32 哪个脚 | 方向 | 备注 |
| --- | --- | --- | --- | --- | --- |
| VCC | 电源 | 3.3V | 3.3V | 电源 | 先确认模块是否支持 3.3V |
| GND | 地 | 0V | GND | 电源 | 必须共地 |
| TX | 模块发送 | 3.3V TTL | STM32 RX | 模块 -> STM32 | 串口要交叉 |
| RX | 模块接收 | 3.3V TTL | STM32 TX | STM32 -> 模块 | 可能暂时不用 |

## 4. 本项目推荐的实际接线

当前已有示例工程采用：

- 北斗/GPS 模块接 `USART2`。
- OLED 使用 7 针 SPI 屏，但代码中用 GPIO 模拟 SPI。
- ST-LINK 使用 SWD 下载和调试。

### 4.1 北斗/GPS 模块接线

北斗/GPS 模块通过 UART 输出 NMEA 数据。

推荐接 `USART2`：

| STM32F103C8T6 小系统板 | 北斗/GPS 模块 | 方向 | 说明 |
| --- | --- | --- | --- |
| `3.3V` | `VCC` | 供电 | 先确认模块支持 3.3V 供电 |
| `GND` | `GND` | 共地 | 必须连接 |
| `PA2 / USART2_TX` | `RX` | STM32 -> 模块 | STM32 给模块发配置命令，当前可先接上 |
| `PA3 / USART2_RX` | `TX` | 模块 -> STM32 | 模块输出 NMEA，最关键 |

串口参数：

```text
baudrate: 9600
data:     8 bit
parity:   none
stop:     1 bit
flow:     none
```

为什么 TX/RX 要交叉：

```text
模块 TX 是模块发出的数据，必须接 STM32 RX。
STM32 TX 是 STM32 发出的数据，必须接模块 RX。
```

如果接成 `TX -> TX`、`RX -> RX`，通常收不到数据。

### 4.2 OLED 接线

当前项目使用的是 7 针 SPI OLED，代码里用 GPIO 模拟 SPI，不依赖 STM32 硬件 SPI 外设。

接线：

| STM32F103C8T6 小系统板 | OLED 引脚 | 方向 | 说明 |
| --- | --- | --- | --- |
| `GND` | `GND` | 供电 | 地 |
| `3.3V` | `VDD` | 供电 | OLED 电源 |
| `PB8` | `D0 / SCK` | STM32 -> OLED | SPI 时钟 |
| `PB9` | `D1 / SDA` | STM32 -> OLED | SPI 数据 |
| `R / RESET` | `RES` | 复位 | 接小系统板复位脚 |
| `PB12` | `DC` | STM32 -> OLED | 命令/数据选择 |
| `PB13` | `CS` | STM32 -> OLED | 片选 |

当前代码中的 OLED 引脚定义：

```c
#define OLED_SCK_GPIO_Port GPIOB
#define OLED_SCK_Pin       GPIO_PIN_8

#define OLED_SDA_GPIO_Port GPIOB
#define OLED_SDA_Pin       GPIO_PIN_9

#define OLED_DC_GPIO_Port  GPIOB
#define OLED_DC_Pin        GPIO_PIN_12

#define OLED_CS_GPIO_Port  GPIOB
#define OLED_CS_Pin        GPIO_PIN_13
```

注意：

- 7 针 SPI OLED 和 4 针 I2C OLED 不是一回事。
- 如果你的 OLED 只有 `GND/VCC/SCL/SDA` 四个脚，那是 I2C OLED，接线和代码都要改。
- 如果 OLED 控制器不是 SSD1306，而是 SH1106，可能会出现显示偏移，需要改 OLED 驱动。

### 4.3 ST-LINK 接线

ST-LINK 用于烧录和调试，不属于业务功能模块，但开发时必须准备。

常见 SWD 接线：

| ST-LINK | STM32F103C8T6 | 说明 |
| --- | --- | --- |
| `SWDIO` | `SWDIO / PA13` | 调试数据 |
| `SWCLK` | `SWCLK / PA14` | 调试时钟 |
| `GND` | `GND` | 必须共地 |
| `3.3V / VTref` | `3.3V` | 电压参考或供电，按板子实际供电方式决定 |
| `NRST` | `NRST / R` | 可选，建议接，方便复位和连接 |

设计时的原则：

- 不要把 `PA13`、`PA14` 分配给 OLED、北斗等模块。
- CubeMX 里 `SYS -> Debug` 要选择 `Serial Wire`。
- 如果程序把调试引脚占用了，后续可能出现烧录/调试困难。

## 5. 电源设计思路

硬件连接先看电源，再看信号。

### 5.1 本项目推荐电源方案

入门实验可以这样供电：

```text
USB 或 ST-LINK 给 STM32 小系统板供电
STM32 小系统板 3.3V 给北斗/GPS 模块和 OLED 供电
所有 GND 接在一起
```

但要注意：

- 北斗/GPS 模块冷启动、搜星时电流会波动，电源不稳会导致模块反复重启或无数据。
- OLED 虽然电流不大，但也要确认供电稳定。
- 如果模块总电流超过小系统板 3.3V 稳压能力，就要单独加 3.3V 稳压模块。

### 5.2 3.3V 和 5V 的问题

STM32F103C8T6 是 3.3V 单片机。

设计原则：

```text
能用 3.3V 就统一用 3.3V。
不确定模块 IO 是否 5V 容忍时，不要把 5V 信号直接接 STM32。
```

有些模块板写着支持 5V 供电，但它的串口 TX/RX 不一定是 5V 安全。必须看模块说明。

### 5.3 为什么一定要共地

所有通信线都必须有共同参考地。

如果 STM32 和模块没有共地：

- 串口可能乱码。
- OLED 可能不显示。
- ST-LINK 可能连接不稳定。
- 信号高低电平没有共同参考，通信不可预测。

结论：

```text
所有模块 GND 必须连在一起。
```

## 6. 为什么这样分配接口

### 6.1 北斗/GPS 用 UART

北斗/GPS 模块输出的是 NMEA 文本数据，本质是一串 ASCII 字符。

典型内容：

```text
$GNRMC,hhmmss.sss,A,ddmm.mmmm,N,dddmm.mmmm,E,...
```

这种数据最适合用 UART 接收。

使用 `USART2` 的好处：

- `PA2/PA3` 是 F103 常用串口引脚。
- 不占用 `PA9/PA10`，后面可以把 USART1 留给日志或 USB 转串口调试。
- 和当前已有工程一致，减少修改。

### 6.2 OLED 用 GPIO 模拟 SPI

OLED 显示数据量不大，刷新速度要求不高。

用 GPIO 模拟 SPI 的好处：

- 不要求接到 STM32 硬件 SPI 的固定引脚。
- 当前接线已经固定在 `PB8/PB9/PB12/PB13`。
- 程序更容易和现有资料对齐。

缺点：

- 速度不如硬件 SPI。
- 驱动代码要自己控制 SCK/SDA 时序。

对于 128x64 OLED 显示经纬度，这个速度足够。

### 6.3 ST-LINK 用 SWD

ST-LINK 通过 SWD 连接 STM32，可以完成：

- 烧录。
- 复位。
- 单步调试。
- 查看变量和寄存器。

比串口下载更适合开发阶段。

## 7. CubeMX 中需要怎么配置

硬件接线和 CubeMX 配置必须一致。

### 7.1 SYS

```text
SYS -> Debug -> Serial Wire
```

目的：

- 保留 SWD 调试接口。
- 避免 PA13/PA14 被当成普通 GPIO。

### 7.2 USART2

配置：

```text
USART2 -> Mode -> Asynchronous
Baud Rate -> 9600
Word Length -> 8 Bits
Parity -> None
Stop Bits -> 1
Hardware Flow Control -> None
```

引脚：

```text
PA2 -> USART2_TX
PA3 -> USART2_RX
```

中断：

```text
NVIC -> USART2 global interrupt -> Enable
```

原因：

- 北斗/GPS 模块会持续输出 NMEA 数据。
- 用中断接收比主循环轮询更可靠。

### 7.3 OLED GPIO

如果让 CubeMX 管理 OLED 引脚，可以配置：

```text
PB8  -> GPIO_Output
PB9  -> GPIO_Output
PB12 -> GPIO_Output
PB13 -> GPIO_Output
```

输出类型：

```text
Output Push Pull
```

速度：

```text
High 或 Medium
```

当前已有工程中，OLED 驱动 `OLED_Init()` 自己初始化 GPIOB，所以 `.ioc` 里不一定能看到这些 GPIO 配置。后续如果要更规范，可以把这些引脚也放进 CubeMX 管理。

### 7.4 RCC 时钟

当前已有工程使用 HSI 8 MHz，无 PLL，也能完成串口和 OLED 显示。

如果你确认小系统板有 8 MHz 外部晶振，可以后续改成：

```text
RCC -> HSE -> Crystal/Ceramic Resonator
SYSCLK -> 72 MHz
```

但初学阶段不建议一上来就改复杂时钟。硬件先跑通最重要。

## 8. 分步验证流程

不要把所有模块一次性接上然后直接调完整项目。应该分阶段验证。

### 8.1 第 1 步：验证 STM32 最小系统

只接：

- STM32 小系统板。
- ST-LINK。

目标：

- CubeProgrammer 能连接芯片。
- 能烧录 LED 闪烁程序。
- 能复位运行。

不通过就先别接北斗和 OLED。

### 8.2 第 2 步：单独验证 OLED

接：

- STM32。
- OLED。
- ST-LINK。

目标：

- OLED 能清屏。
- OLED 能显示固定英文，例如 `OLED OK`。
- 再显示中文或图形。

排查顺序：

1. VDD/GND。
2. SCK/SDA。
3. DC/CS。
4. RES。
5. OLED 类型是否 SSD1306。

### 8.3 第 3 步：单独验证北斗/GPS 模块

最好先不用 STM32，先用 USB 转 TTL 模块接电脑测试。

接线：

```text
USB-TTL RX -> GPS TX
USB-TTL TX -> GPS RX
USB-TTL GND -> GPS GND
3.3V -> GPS VCC
```

串口工具设置：

```text
9600 8N1
```

目标：

- 能看到 `$GNRMC`、`$GPRMC`、`$BDRMC` 等 NMEA 文本。
- 即使没有定位，也应该有持续输出。

如果电脑串口都看不到数据，不要急着改 STM32 程序，先解决模块供电、天线、波特率、接线问题。

### 8.4 第 4 步：STM32 接收北斗/GPS 数据

接：

- STM32。
- 北斗/GPS。
- ST-LINK。

目标：

- USART2 能收到字节。
- 能捕获完整 `$...` NMEA 帧。
- 能识别 RMC 语句。

如果没有显示屏，也可以先通过调试器看缓冲区，或者再加一个串口日志。

### 8.5 第 5 步：整机联调

最后再同时接：

- STM32。
- 北斗/GPS。
- OLED。
- ST-LINK。

目标：

- 上电后 OLED 显示 `BDS WAITING`。
- 收到 RMC 后显示 `BDS NO FIX` 或 `BDS FIX`。
- 定位成功后显示时间、经纬度。

## 9. 常见故障和排查

### 9.1 OLED 不亮

优先检查：

1. OLED VDD 是否接 3.3V。
2. OLED GND 是否接 GND。
3. `PB8 -> D0/SCK` 是否正确。
4. `PB9 -> D1/SDA` 是否正确。
5. `PB12 -> DC` 是否正确。
6. `PB13 -> CS` 是否正确。
7. `RES` 是否接复位。
8. OLED 是 SPI 还是 I2C。
9. 控制器是否 SSD1306。

### 9.2 一直接收不到北斗数据

检查：

1. GPS 模块是否供电。
2. GPS GND 是否和 STM32 共地。
3. GPS TX 是否接 STM32 PA3。
4. 串口波特率是否 9600。
5. GPS 模块是否有天线。
6. 模块是否实际输出 NMEA。

最常见错误：

```text
TX/RX 没有交叉。
```

### 9.3 一直显示无定位

如果已经能收到 RMC，但状态是 `V`，说明模块还没有定位成功。

处理：

- 把天线放到窗边或室外。
- 等待几分钟到十几分钟。
- 确保天线接口接牢。
- 确认供电稳定。
- 避免在室内深处测试。

### 9.4 ST-LINK 无法连接

检查：

1. ST-LINK 驱动是否正常。
2. SWDIO/SWCLK 是否接对。
3. GND 是否共地。
4. 目标板是否供电。
5. CubeMX 是否保留 `Serial Wire`。
6. BOOT0 是否处于正常状态。
7. 尝试降低 SWD 频率。

### 9.5 程序烧录成功但没有任何反应

可能原因：

- 时钟配置错误。
- OLED 接线错误。
- GPS 模块没有输出。
- 程序卡在初始化。
- 电源不稳。
- 接错了 OLED 类型。

排查方法：

```text
先跑 LED
再跑 OLED 固定显示
再跑 GPS 串口接收
最后整合
```

## 10. 如果重新设计引脚，应该遵守哪些原则

如果不沿用当前接线，重新设计时遵守这些规则。

### 10.1 不占用调试引脚

避免使用：

```text
PA13 SWDIO
PA14 SWCLK
NRST
BOOT0
```

除非你很清楚后果。

### 10.2 串口优先分配给 GPS

GPS 模块持续输出数据，推荐使用硬件 UART：

| 串口 | 常见引脚 | 适合用途 |
| --- | --- | --- |
| USART1 | PA9/PA10 | 日志、调试串口、备用 |
| USART2 | PA2/PA3 | GPS/北斗，当前项目推荐 |
| USART3 | PB10/PB11 | 后续扩展 |

不要用普通 GPIO 硬模拟 GPS 串口，没必要。

### 10.3 OLED 可以选 SPI 或 I2C

如果是 7 针 SPI OLED：

- SCK。
- SDA/MOSI。
- RES。
- DC。
- CS。

如果是 4 针 I2C OLED：

- SCL。
- SDA。

两种屏代码不通用。买屏或接线前一定先看清楚接口。

### 10.4 先留扩展接口

毕业设计后期可能还要接：

- 无线通信模块。
- 蜂鸣器。
- 按键。
- 指示灯。
- 电源检测。
- 传感器。

所以不要一开始把所有方便的引脚都占满。

## 11. 推荐画一张连接表

最终设计完成后，文档里必须保留一张接线总表。

当前项目推荐总表：

| 功能 | STM32 引脚/接口 | 外部模块引脚 | 协议/类型 | 说明 |
| --- | --- | --- | --- | --- |
| GPS 供电 | `3.3V` | `VCC` | 电源 | 确认模块支持 3.3V |
| GPS 地 | `GND` | `GND` | 电源 | 必须共地 |
| GPS 发送到 STM32 | `PA3 / USART2_RX` | `TX` | UART | 接收 NMEA |
| STM32 发送到 GPS | `PA2 / USART2_TX` | `RX` | UART | 可用于配置模块 |
| OLED 电源 | `3.3V` | `VDD` | 电源 | 3.3V |
| OLED 地 | `GND` | `GND` | 电源 | 必须共地 |
| OLED 时钟 | `PB8` | `D0 / SCK` | GPIO 模拟 SPI | 软件 SPI |
| OLED 数据 | `PB9` | `D1 / SDA` | GPIO 模拟 SPI | 只写 |
| OLED 复位 | `R / RESET` | `RES` | 复位 | 接系统复位 |
| OLED 命令/数据 | `PB12` | `DC` | GPIO | 0 命令，1 数据 |
| OLED 片选 | `PB13` | `CS` | GPIO | 低有效 |
| ST-LINK 数据 | `PA13 / SWDIO` | `SWDIO` | SWD | 下载/调试 |
| ST-LINK 时钟 | `PA14 / SWCLK` | `SWCLK` | SWD | 下载/调试 |
| ST-LINK 地 | `GND` | `GND` | 电源参考 | 必须共地 |
| ST-LINK 复位 | `NRST` | `NRST` | 复位 | 建议连接 |

## 12. 设计检查清单

接线前检查：

- [ ] 每个模块的 VCC 电压确认过。
- [ ] 所有 GND 都会连在一起。
- [ ] STM32 没有接入 5V 信号。
- [ ] GPS TX 接 STM32 RX。
- [ ] GPS RX 接 STM32 TX。
- [ ] OLED 是 SPI 版本，不是 I2C 版本。
- [ ] OLED 引脚和代码定义一致。
- [ ] ST-LINK 的 SWDIO/SWCLK 没有被其他模块占用。
- [ ] CubeMX 中 `SYS -> Debug` 是 `Serial Wire`。
- [ ] USART2 配置为 9600 8N1。
- [ ] USART2 中断打开。
- [ ] 天线已经接到北斗/GPS 模块。

上电前检查：

- [ ] 用万用表确认没有 VCC 和 GND 短路。
- [ ] 先不插所有模块，分模块上电。
- [ ] 不确定电压时先不要接 STM32 IO。
- [ ] 杜邦线颜色尽量固定：红色 VCC，黑色 GND。

验证顺序：

- [ ] STM32 + ST-LINK 能连接。
- [ ] STM32 能烧录 LED 程序。
- [ ] OLED 能显示固定内容。
- [ ] GPS 模块能用 USB-TTL 输出 NMEA。
- [ ] STM32 能通过 USART2 接收 GPS 数据。
- [ ] OLED 能显示 GPS 状态。
- [ ] 室外或窗边能定位成功。

## 13. 本项目硬件设计结论

本项目当前阶段最合理的硬件方案是：

```text
STM32F103C8T6 作为主控
USART2 接北斗/GPS 模块
GPIO 模拟 SPI 接 7 针 OLED
ST-LINK 通过 SWD 下载和调试
所有模块统一 3.3V 供电并共地
```

这样设计的优点：

- 模块分工清楚。
- 接线数量可控。
- 不占用 SWD 调试接口。
- GPS 使用硬件串口，接收可靠。
- OLED 使用模拟 SPI，和现有代码一致。
- 后续还能保留其他串口或 GPIO 扩展无线通信模块。

最重要的设计原则：

```text
硬件连接必须和软件配置一致。
软件里的 USART2、PB8/PB9/PB12/PB13，不是随便写的，它们对应真实接线。
```

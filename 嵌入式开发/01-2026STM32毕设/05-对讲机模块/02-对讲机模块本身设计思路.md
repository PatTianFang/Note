# 02-对讲机模块本身设计思路

## 1. 低频、中频、高频的划分结论

这里的“低频、中频、高频”不能按普通电源电路的频率理解，而要按对讲机无线收发链路来划分：

| 分类 | 总原理图中能看到的部分 | 主要元件/网络 | 能否从当前资料画出完整电路 |
| --- | --- | --- | --- |
| 高频部分 | 天线接口和射频匹配网络 | `ANT`、`ANT1`、`RF`、`L2`、`L3`、`C4`、`C7`、`C8`、`C9`、`C16`、`R8`、对讲模块 `ANT` 引脚 | 可以画出演示板外部的天线匹配电路 |
| 中频/基带部分 | 对讲模块内部的接收、发射、调制解调、中频滤波等核心电路 | PDF 中的对讲模块 `U3`，ASC 文件中对应 `BK1` 模块焊盘/连接器；外部只露出 `RXD`、`TXD`、`PTT`、`SQ`、`PD`、`HML`、`VOX` 等接口 | 不能画成元件级中频电路，只能把模块画成黑盒 |
| 低频部分 | 麦克风输入、外部音频输入、模块音频输出、音频功放、喇叭 | `MIC1`、`MIC_IN`、`LINE_IN`、`AFOUT/AF_OUT`、`U2`、`SPK`、`R4`、`R9`、`C11`、`C12`、`C3`、`C23`、`R10`、`R11`、`C24`、`C26`、`C32` 等 | 可以画出演示板外部的音频输入/输出电路 |
| 电源与控制辅助 | 不是低中高频本身，但三部分都必须依赖 | `VBAT`、`+3V`、`GND`、`PD`、`PTT`、`SQ`、`RXD/TXD`、`HML`、`D1`、`R3` 等 | 应单独画成供电和控制接口图 |

最重要的边界是：**当前总原理图没有给出对讲模块内部芯片的中频混频器、中频滤波器、鉴频/调制、功放驱动等细节**。所以不能为了凑“低中高频三个分图”而自行编造中频电路。正确做法是：低频和高频可以按原理图外部元件画；中频/基带只能画成 `U3/BK1` 模块内部功能框图，并标注“内部资料缺失”。

## 2. 总原理图中各部分分别在哪里

### 2.1 高频部分：天线和射频匹配

高频部分从对讲模块的 `ANT` 射频脚出来，经过一组 LC 匹配/滤波元件，再到天线座 `ANT`。在 ASC 文件里，主要网络是：

```text
对讲模块 ANT 脚/BK1.7 -> RF -> L3/C9/C16 -> L2/C7/C8 -> ANT1 -> ANT 天线座
```

对应元件：

| 元件/网络 | 作用 | 设计和连接考量 |
| --- | --- | --- |
| `ANT` | 天线座或天线焊点 | 天线是射频能量真正发射/接收的位置，必须匹配模块工作频段。说明书提到 `SR_FRS_2WUS` 属于 `400M-480M` 系列，天线谐振频率必须和实际设置频率一致 |
| `ANT1` | 天线端射频网络 | 连接天线座和匹配网络，是高频信号路径的一部分。走线应短、直、远离数字噪声 |
| `RF` | 模块端射频网络 | 连接对讲模块 `ANT` 引脚和外部匹配网络。这里不能接普通 GPIO、串口或音频线 |
| `L2`、`L3` | 串联或匹配电感 | 和周围小电容组成阻抗匹配网络，用来把模块射频口、PCB 走线、天线之间的阻抗调到合适范围。ASC 中标注为线圈规格 `0.35x1.5x4T`，不能随意替换 |
| `C7`、`C8`、`C9`、`C16` | pF 级射频电容 | 和 `L2/L3` 配合完成匹配、滤波、谐波抑制。ASC 中可见 `C7=6P`、`C8=2P`、`C9=2P5`、`C16=4P`；实际值应以实物和 BOM 为准 |
| `C4` | 预留/可调匹配电容位 | ASC 中标注 `NC`，表示可能默认不焊，用于调试天线匹配 |
| `R8` | 射频端预留/泄放/匹配相关位置 | ASC 中标注 `3K3`，一端接 `ANT1`，另一端接地。具体用途需要结合厂家设计说明确认，不能只凭当前资料断定为某一种射频功能 |
| `GND` | 高频参考地 | 高频电路不是只看一根信号线，回流地同样属于电路。天线座地、匹配电容地、模块地必须低阻连接 |

这部分画分图时，要保留模块 `ANT` 引脚、匹配网络、天线座和地。不要把射频匹配元件当成普通滤波电容随便删除。

### 2.2 中频/基带部分：主要在对讲模块内部

对讲机真正的中频处理通常包括接收混频、中频滤波、限幅/鉴频、发射调制、频率合成、收发切换等。当前资料没有把这些电路展开，而是封装在对讲模块内部。

总原理图外部能看到的是模块接口：

| 接口 | 方向 | 说明 |
| --- | --- | --- |
| `RXD` / `TXD` | 外部 MCU 与模块串口通信 | 说明书写明为 `3V TTL` 串口，演示参数为 `9600 8N1`，可用于参数设置和短数据传输 |
| `PTT` | 外部 -> 模块 | 低电平发射，高电平或悬空接收 |
| `SQ` / `SPK_EN` | 模块 -> 外部 | 低电平表示收到对方发射信号；也可控制音频功放静噪 |
| `PD` | 外部 -> 模块 | 说明书明确要求给高电平才能进入工作状态 |
| `HML` / `H/L` | 外部 -> 模块 | 高低功率选择；说明书强调不能给高电平，只能悬空或拉低 |
| `VOX` | 模块功能脚 | 当前资料只在原理图引脚中出现，缺少完整使用说明 |
| `MIC_IN`、`AFOUT` | 模块音频输入/输出 | 属于低频音频接口，但进入模块后会参与内部调制/解调链路 |
| `ANT` | 模块射频输入/输出 | 属于高频接口，进入模块后连接内部收发机链路 |

因此，中频分图建议画成下面这种功能框图，而不是画元件级电路：

```text
                 RXD/TXD 参数设置和数据传输
                         |
MIC_IN/LINE_IN -> [ 对讲模块 U3/BK1 内部：调制、混频、中频/基带、功率控制 ] -> ANT/RF
                         |
AFOUT       <- [ 接收解调、音频恢复、静噪判断 SQ ]
```

这张图的设计含义是说明“模块内部完成无线对讲核心收发”，不是说明已经知道内部每个中频元件。若论文或答辩中被问到“中频电路怎么画”，应回答：**当前资料没有模块内部原理图，所以只能画模块级框图；若要画真实中频电路，需要厂家提供模块芯片方案或内部原理图。**

### 2.3 低频部分：麦克风输入、音频输出和功放

低频部分可以分成发送音频输入和接收音频输出两条线。

发送方向：

```text
MIC1 / LINE_IN -> 偏置、耦合、滤波、限幅/衰减 -> MIC_IN -> 对讲模块内部调制
```

接收方向：

```text
对讲模块 AFOUT/AF_OUT -> 耦合/增益/滤波 -> U2 音频功放 -> SPK 喇叭
```

说明书明确写了两个关键点：

- `AF_OUT` 是模块音频直接输出，最大约 `250 mV`，没有经过音频功放。
- `Line_In` 是外接音源输入，信号必须衰减到 `30 mV` 以内。

所以低频分图必须把“小信号音频”和“功放后的喇叭输出”分清楚。`Line_In/MIC_IN` 不能直接接 STM32 的 UART 文本，也不能直接接 3.3 V 方波；如果要做 GPS 语音播报，必须先有 TTS 或音频生成电路，再把模拟音频衰减、耦合后送入 `Line_In/MIC_IN`。

## 3. 怎么通过总原理图画出低、中、高频三个电路

从总原理图拆分小电路时，不是按图纸上的位置框选，而是按“网络名”和“信号流向”拆分。

### 3.1 第一步：先找核心模块边界

先找到对讲模块本体。PDF 中可按 `U3` 理解，ASC 文件中模块焊盘/连接器对应 `BK1`。它是三个频段的中心边界：

- `ANT` 脚向外是高频射频匹配和天线。
- `MIC_IN`、`AFOUT` 向外是低频音频输入/输出。
- `RXD/TXD/PTT/SQ/PD/HML/VOX` 是控制、串口和状态接口。
- 中频/基带在模块内部，外部图纸不展开。

### 3.2 第二步：按网络名追线

建议在原理图或 ASC 网络表中按这些名字搜索：

| 要画的分图 | 重点搜索的网络名 |
| --- | --- |
| 高频射频图 | `ANT`、`ANT1`、`RF`、`GND` |
| 低频音频图 | `MIC_IN`、`LINE_IN`、`AFOUT`、`AF_OUT`、`SPK0`、`SPK1B`、`SQ` |
| 中频/基带框图 | `RXD`、`TXD`、`PTT`、`SQ`、`PD`、`HML`、`VOX`、`ANT`、`MIC_IN`、`AFOUT` |

追线时，凡是同一个网络名，即使在图上没有画一根连续导线，也表示电气相连。

### 3.3 第三步：每张分图只保留必要元件

拆图时可以删掉与本功能无关的元件，但不能删掉电源和地参考。

高频图保留：

```text
U3/BK1 ANT 引脚 -> L2/L3/C4/C7/C8/C9/C16/R8 -> ANT 天线座 -> GND
```

低频图保留：

```text
MIC1/LINE_IN -> R4/R9/C11/C12/C3 等输入网络 -> MIC_IN -> U3/BK1
U3/BK1 AFOUT -> C23/R10/R11/C24/C26 等网络 -> U2 -> SPK
SQ -> R1/U2 控制或静噪相关引脚
```

中频图保留：

```text
U3/BK1 模块黑盒
外部接口：RXD、TXD、PTT、SQ、PD、HML、VOX、MIC_IN、AFOUT、ANT、VBAT、GND
内部标注：射频收发、调制/解调、中频/基带、静噪判断
```

### 3.4 第四步：按信号方向重画

建议分图都按从左到右的信号流向画：

- 高频图：模块 `ANT` 在左，天线 `ANT` 在右，匹配网络放中间。
- 低频发送图：麦克风或外部音源在左，模块 `MIC_IN` 在右。
- 低频接收图：模块 `AFOUT` 在左，功放 `U2` 和喇叭在右。
- 中频图：模块画在中间，左侧放音频/串口输入，右侧放射频和音频输出，控制脚放下方。

这样画出来的图比直接裁剪总原理图更适合说明“这个电路在系统里干什么”。

## 4. 各部分具体设计思路和元件连接考量

### 4.1 高频射频匹配电路

高频部分的设计目标不是“把模块脚直接接天线”，而是让模块射频口、PCB 走线和天线之间尽量匹配，减少反射和损耗。

推荐分图表达：

```text
U3/BK1 ANT
   |
  RF
   |
 L3 / C9 / C16
   |
 L2 / C7 / C8 / C4
   |
 ANT1
   |
 ANT 天线座
```

设计考量：

| 元件 | 连接方式 | 设计考量 |
| --- | --- | --- |
| `L2`、`L3` | 串在射频主路径或参与匹配网络 | 用于调节射频阻抗。对讲模块工作在百 MHz 级，电感封装、线圈尺寸、焊盘寄生参数都会影响结果，所以不能像普通低频电感一样随意改值 |
| `C7`、`C8`、`C9`、`C16` | 多数为 pF 级到地或跨节点连接 | 和电感组成 π 型或多级匹配/滤波结构，用于阻抗变换和高频滤波。pF 级电容的实际效果受 PCB 寄生电容影响很大 |
| `C4` | 预留不焊或调试位 | 给天线匹配调试留余量。不同外壳、天线、频点可能需要改匹配值 |
| `R8` | 天线端到地相关支路 | ASC 只显示其连接和 `3K3` 标注，缺少厂家说明，不能断定其唯一功能；画图时保留即可 |
| `ANT` | 连接外部天线 | 天线必须匹配实际工作频率。说明书明确提醒不要随便拿未知谐振频率天线使用，否则会影响发射功率和接收灵敏度 |
| 高频 `GND` | 匹配电容和天线座地回流 | 高频回流路径必须短，匹配元件附近应有完整地参考。画 PCB 时要避免射频线穿过分割地或绕远路 |

设计时的关键限制：

1. 高频走线尽量短、直、少过孔。
2. 匹配网络靠近模块射频脚和天线座布置。
3. 不要让 STM32 时钟线、OLED 通信线、DCDC 开关节点靠近射频走线。
4. 匹配值需要实测调试，当前资料不足以证明这些值适合所有外壳和天线。

### 4.2 中频/基带与控制接口

中频/基带部分的设计目标是让外部 MCU 能控制模块工作状态，同时不干扰模块内部无线收发。

推荐分图表达：

```text
                 RXD  <---- STM32/USB-TTL TX
                 TXD  ----> STM32/USB-TTL RX
                 PTT  <---- 按键或 MCU 控制，低电平发射
                 PD   <---- 上电使能，高电平工作
                 HML  <---- 悬空或拉低，不允许强推高电平
                 SQ   ----> 接收信号指示，低电平有效

        [ U3/BK1 对讲模块内部中频/基带/调制解调 ]

MIC_IN/LINE_IN ----> 内部发射调制
AFOUT       <---- 内部接收解调音频
ANT/RF      <----> 内部射频收发
```

元件和连接考量：

| 接口/元件 | 连接方式 | 设计考量 |
| --- | --- | --- |
| `RXD/TXD` | 交叉接 USB-TTL 或 STM32 UART | 说明书写明是 `3V TTL`，不能用 5 V 串口直接接。若接 STM32F103 的 3.3 V UART，仍应向厂家确认 3.3 V 输入是否长期允许 |
| `PTT`、`S2` | 按键或 MCU 拉低控制 | 语音对讲时低电平发射，高电平或悬空接收。若由 STM32 控制，推荐用开漏或三极管/MOS 只负责拉低，避免误推高电平 |
| `R12`、`R7`、`C5` | PTT 相关上拉、限流、滤波/消抖网络 | 这些元件让按键或控制信号更稳定，减少误触发。当前资料可确认它们在 PTT 附近，但具体阈值和延时需以实物测试为准 |
| `SQ`、`R1` | 模块接收状态输出，关联音频功放静噪 | 说明书说 `SQ` 高电平表示没收到信号，低电平表示收到信号；可用于打开/关闭功放，避免无信号时喇叭噪声 |
| `PD`、`R2` | 模块工作使能 | 说明书明确 `PD` 必须给高电平才能进入工作状态。连接 STM32 时要确认默认上电状态，避免模块无法启动 |
| `HML/H/L` | 高低功率选择 | 说明书强调不能给高电平，只能悬空或拉低。若接 MCU，必须保证复位阶段也不会输出高电平 |
| `VOX` | 语音触发相关接口 | 当前资料没有给出完整 VOX 使用方式，不能作为确定设计依据 |

这里的“中频设计”本质上是模块选型和接口设计，不是离散中频电路设计。若需要画到混频器、中频滤波器、鉴频器一级，必须向厂家索要模块内部原理图或芯片资料。

### 4.3 低频发送音频输入电路

发送音频输入负责把麦克风或外部音源变成模块可接受的小信号音频。

推荐分图表达：

```text
MIC1
  |
R4 / R9 偏置与限流
  |
C11 / C12 / C3 耦合与滤波
  |
MIC_IN
  |
U3/BK1 对讲模块

外部 Line_In 音源 -> 衰减到 30 mV 以内 -> MIC_IN/模块音频输入
```

元件和连接考量：

| 元件/网络 | 连接方式 | 设计考量 |
| --- | --- | --- |
| `MIC1` | 板载麦克风 | 提供人声输入。麦克风信号是毫伏级小信号，容易受电源噪声和射频串扰影响 |
| `R4` | 麦克风相关偏置/限流电阻，ASC 中标注 `680R` | 给麦克风或输入网络提供合适工作条件，限制电流，避免直接硬接到模块输入 |
| `R9` | 输入网络电阻，ASC 中标注 `1K8` | 和电容共同影响输入阻抗、增益和滤波特性 |
| `C11` | 耦合/滤波电容，ASC 中标注 `1U` | 隔直流、保留音频交流分量，避免前后级直流偏置互相影响 |
| `C12` | 去耦/滤波电容，ASC 中标注 `100N` | 抑制高频噪声，让麦克风输入更稳定 |
| `C3` | 小电容滤波，ASC 中出现 `470P/1N` 标注不一致 | 用于抑制射频或高频尖峰进入音频输入。取值应以实物 BOM 为准 |
| `MIC_IN` | 模块音频输入脚 | 只能接小信号模拟音频，不能直接接 GPS 串口、STM32 UART 或 3.3 V GPIO |
| `LINE_IN` | 外部音源输入 | 说明书要求信号衰减到 `30 mV` 以内。若用 TTS 模块输出，必须先做衰减、耦合和必要滤波 |

设计时要注意：

1. 麦克风线和 `MIC_IN` 线远离天线和 DCDC 开关节点。
2. 外部音源必须先确认幅度，不能把手机耳机口、功放输出、STM32 PWM 原始方波直接接入。
3. 若目标是 GPS 语音播报，STM32 必须先驱动 TTS 或音频生成模块，把文本变成模拟语音，再接 `Line_In`。

### 4.4 低频接收音频输出和功放电路

接收音频输出负责把模块解调出来的音频放大到能推动喇叭。

推荐分图表达：

```text
U3/BK1 AFOUT
   |
C23 耦合
   |
R10 / R11 / C24 增益与滤波网络
   |
U2 音频功放
   |
C32 / C33 / C41 / C45 / C2 去耦与稳定
   |
SPK 喇叭

SQ -> U2 静噪/使能相关控制
```

元件和连接考量：

| 元件/网络 | 连接方式 | 设计考量 |
| --- | --- | --- |
| `AFOUT/AF_OUT` | 模块音频输出 | 说明书写明最大约 `250 mV`，未经过音频功放。它适合送入后级功放或外部音频处理，不适合直接推喇叭 |
| `C23` | `AFOUT` 后的耦合电容，ASC 中标注 `1UF` | 隔离模块输出直流偏置，只让音频交流信号进入功放输入网络 |
| `R10` | 功放输入/反馈相关电阻，ASC 中标注 `220K` | 和 `R11/C24` 一起决定输入阻抗、增益或频响，不能随意改动 |
| `R11` | 功放输入/反馈相关电阻，ASC 中标注 `22K` | 和 `R10` 的比例可能影响放大倍数，需结合 `U2` 实际型号数据手册确认 |
| `C24` | 小电容，ASC 中标注 `100P` | 用于高频补偿或抑制尖峰，避免功放自激和射频串入 |
| `U2` | 音频功放 | PDF/ASC 对型号标注不完全一致：有 `LM4871`，也出现 `MC34119D/LM4890` 字样。正式说明应写“音频功放 U2”，具体型号以实物 BOM 为准 |
| `C26` | 功放相关耦合/旁路电容，ASC 中标注 `1uf` | 用于稳定功放偏置或交流通路，具体作用需结合 U2 实际型号引脚定义确认 |
| `C32` | 电源大电容，ASC 中标注 `220uF/10V` | 为功放瞬态电流提供缓冲，减少喇叭大音量时电源跌落 |
| `C33`、`C41`、`C45` | 小电容，ASC 中标注 `100P` | 抑制高频噪声、改善功放稳定性，避免射频信号进入音频功放 |
| `C2` | 电源滤波电容，ASC 中标注 `100uF/10V` | 提供低频电源滤波，减小音频噪声和电源波动 |
| `SPK` | 喇叭接口，ASC 中标注 `8R/0.5W` | 功放输出接喇叭，阻抗和功率要匹配，不能把模块 `AFOUT` 直接当作喇叭输出 |
| `SQ` / `R1` | 静噪控制相关 | 无接收信号时关闭或静音功放，减少底噪和耗电；有接收信号时打开功放播放声音 |

设计时要注意：

1. 功放电源要有足够电流和本地去耦，否则喇叭声音容易失真或引入啸叫。
2. 喇叭线不要贴近麦克风输入线，避免声电反馈。
3. 若外接更大功率功放，说明书建议可以从 `AF_OUT + GND` 取未功放音频，或从喇叭输出经电容耦合取较大幅度音频；两种方式不能混淆。

### 4.5 电源和指示电路

电源不是低中高频的一部分，但它直接决定对讲模块能否稳定发射。

| 元件/网络 | 连接方式 | 设计考量 |
| --- | --- | --- |
| `J4` / `VBAT` | 外部供电输入 | 说明书推荐 `3.6 V - 4.5 V`，推荐 `4.2 V`。发射电流较大，不建议直接从 STM32 小板 3.3 V 取电 |
| `C1`、`C13` | 模块电源滤波 | 大小电容组合，用于吸收发射瞬态电流和高频噪声 |
| `D1`、`R3` | 电源或状态指示 LED 相关 | `R3` 限制 LED 电流，避免 LED 直接接电源烧毁 |
| `R13`、`C10`、`+3V` | 模块提供或使用的 3 V 相关网络 | 资料里有“模块自身向外提供 +3V”这类设计含义，但当前文件不足以确认所有使用场景；外部系统不要随意反灌电源 |
| `GND` | 全系统公共地 | STM32、GPS、TTS/音频模块、对讲模块必须共地，否则串口、电平控制和音频输入都没有统一参考 |

设计时要注意：

1. 发射时电流比接收时大，电源要留足余量。
2. DCDC 降压电源必须滤波好，说明书明确说滤波不好会引入干扰并降低接收灵敏度。
3. 若从 5 V 供电，说明书建议串硅二极管降到 4.5 V 以内，但正式设计仍应按实际发射电流、压降和发热校核。

### 5.3 低频音频分图

图名建议：`音频输入输出与功放电路`。

```text

MIC1/LINE_IN -> R4/R9/C11/C12/C3 -> MIC_IN -> U3/BK1
```

- “`Line_In` 输入必须衰减到 30 mV 以内。”
- “`AF_OUT` 是未经过功放的音频输出，最大约 250 mV。”

## 6. 当前资料不足，不能编造的内容

以下内容当前资料没有给出，不能在设计说明里写成已经确定：

1. 对讲模块内部真实中频电路，包括混频器、中频滤波器、鉴频器、调制器、频率合成器、射频功放等元件级连接。

4. `Line_In` 的完整推荐应用电路，包括输入阻抗、内部偏置、建议耦合电容和最大 RMS/峰峰值定义。

6. `PTT` 拉低后到可输入音频之间的厂家推荐等待时间。

## 7. 总结

总原理图中能可靠划分的是：**高频部分是天线和射频匹配网络；低频部分是麦克风、外部音频输入、模块音频输出、功放和喇叭；中频/基带部分在对讲模块内部，外部原理图只能画成模块黑盒和接口功能框图。**

通过总原理图画三个电路时，应按网络名追线，而不是按图纸位置裁剪。`ANT/RF` 追出高频图，`MIC_IN/LINE_IN/AFOUT/SPK` 追出低频图，`RXD/TXD/PTT/SQ/PD/HML/VOX` 和模块黑盒组成中频/控制框图。凡是资料没有展开的模块内部结构，都要明确写“资料不足，不能确认”，不能自行画成确定电路。

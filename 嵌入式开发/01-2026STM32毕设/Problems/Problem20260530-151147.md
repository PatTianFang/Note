# Problem20260530-151147

## 任务目标

为 `01-环境准备/01-环境准备.md` 编写一份面向 STM32 初学者的软件开发前环境准备文档，说明电脑上需要准备什么、下载哪些软件、每个软件的用途、安装顺序、验证方法和常见问题。

## 思考过程

- 用户明确说明“我是小白，什么也不懂”，所以文档不能只列软件名，必须解释每个工具在 STM32 开发链路中的位置。
- 优先选择 ST 官方工具链作为主路线：STM32CubeIDE、STM32CubeMX、STM32CubeProgrammer、ST-LINK 驱动和对应 STM32Cube MCU Package。
- 考虑到项目参考资料中出现 `STM32F103C8T6`，文档中特别补充 `STM32CubeF1`、串口工具、USB 转串口驱动和 RS232/TTL 电平注意事项。
- Keil、VS Code、CMake、Ninja、Arm GNU Toolchain 等工具容易让新手混乱，因此放在可选路线中，而不是作为第一步必装。
- 下载链接采用官方入口，具体版本不写死，避免后续官网更新导致文档过期。

## 遇到的问题

- `01-环境准备` 目录存在，但 `01-环境准备.md` 文件尚未实际落盘，需要新建。
- 当前目录不是 Git 仓库，无法通过 `git status` 跟踪差异。
- 创建 `Problems` 目录时，当前 PowerShell 环境的 `New-Item` 不支持 `-LiteralPath` 参数。

## 解决方法

- 使用 `apply_patch` 新建 `01-环境准备/01-环境准备.md`，保持修改范围只在用户指定文档和任务记录文件内。
- 改用 `New-Item -Path 'Problems' -Force` 创建记录目录。
- 文档以“必装、建议、可选、验证、常见问题”的结构组织，避免一开始堆叠过多专业工具。

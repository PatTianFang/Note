# Problem20260530-162044

## 任务目标

为 `01-环境准备/03-答疑解惑.md` 编写一份 STM32 开发环境答疑文档，逐条回答用户提出的 13 个问题，解释 CubeMX、Keil、CMake、编译、调试、ST-LINK、MCU Package、CubeIDE 与 VS Code、`.bin` 起始地址、ST-LINK USB Driver、CMake/Ninja/Arm GNU Toolchain、外部晶振等概念。

## 思考过程

- 用户的问题体现出对工具链分层不清楚，因此文档需要先建立总流程，再逐条解释。
- 重点不是堆定义，而是说明每个工具为什么存在、没有它会怎样、它和其他工具的边界是什么。
- 对“构建、编译、CMake、Ninja、Arm GNU Toolchain”的关系进行拆分，避免误以为 CMake 是编译器。
- 对“调试是否要连接单片机、是否要烧录”进行硬件运行角度解释，说明 PC 不能直接运行访问 STM32 外设寄存器的程序。
- 对外部晶振部分从 HSI/HSE/LSI/LSE、怎么看实物、怎么看原理图、CubeMX 怎么选、配错会怎样几个角度展开。

## 遇到的问题

- `03-答疑解惑.md` 文件存在但内容为空，需要完整新建。
- 用户一次提出 13 个问题，涉及概念跨度大，如果按单个定义回答会割裂，需要先用总流程串起来。
- STM32CubeIDE for Visual Studio Code 插件属于当前 ST 官方工具链的一部分，文档需要避免写成桌面版 STM32CubeIDE。

## 解决方法

- 使用“总流程 + 逐条问答 + 总结表 + 资料入口”的结构。
- 对每个问题按“是什么、为什么、没有会怎样、和其他工具关系”组织。
- 保留面向 F103C8T6 的具体例子，例如 `STM32CubeF1`、`PC13`、`0x08000000`、HSE 8 MHz 常见情况。

# Problem20260530-172032

## 任务目标

为 `03-硬件连接/01-总体思路.md` 编写 STM32 + 北斗定位 + OLED 显示屏硬件连接总体设计文档，说明硬件连接前如何设计、如何分配接口、如何接线、如何配置 CubeMX、如何分步验证和排查问题。

## 思考过程

- 用户问的是“应该怎么设计”，不是只要一张接线表，因此文档需要先讲模块分工、设计顺序、电源/电平/协议/引脚方向，再给实际接线。
- 项目现有参考工程已经明确使用 STM32F103C8T6、USART2 接北斗/GPS、7 针 SPI OLED、PB8/PB9/PB12/PB13 GPIO 模拟 SPI，因此文档应优先沿用已有工程，而不是重新发明接线。
- 硬件设计最容易出错的地方是电源、电平、共地、TX/RX 方向、OLED 接口类型、ST-LINK 调试引脚冲突，需要重点强调。
- 为了降低联调难度，文档采用“STM32 最小系统 -> OLED 单测 -> GPS 单测 -> STM32 接收 GPS -> 整机联调”的验证路径。

## 遇到的问题

- `03-硬件连接/01-总体思路.md` 文件存在但为空，需要完整新建。
- 资料中同时存在 OLED 示例、北斗/GPS 模块资料和已有 VSCode 工程，需要从实际工程中提取可靠接线。
- OLED 可能有 SPI/I2C 不同版本，必须在文档中提醒不能混用代码和接线。

## 解决方法

- 读取已有 `README.md`、`oled_spi.h`、`oled_spi.c`、`bds_nmea` 和 `main.c`，确认实际使用 USART2、PB8/PB9/PB12/PB13、9600 8N1、RMC 解析等信息。
- 编写总体连接框图、接线表、CubeMX 配置表、分步验证流程和常见故障排查。
- 明确本项目推荐方案为 STM32F103C8T6 + 北斗/GPS UART + 7 针 SPI OLED + ST-LINK SWD。

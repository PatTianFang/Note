# Problem20260530-181216

## 任务目标

为 `03-硬件连接/00-硬件连接设计方法.md` 编写一份通用硬件连接设计方法论文档，不绑定具体项目，说明进行 STM32 硬件连接时应该怎么设计、如何考虑、查阅哪些资料、遵循什么思想和流程。

## 思考过程

- 用户明确要求“不要具体这个项目的”，因此文档不能使用北斗/OLED 的具体接线作为主体，而应抽象成通用流程和检查清单。
- 硬件连接的核心风险通常来自电源、电平、通信协议、引脚冲突、调试接口占用、上拉/下拉、分阶段验证不足，因此文档围绕这些维度展开。
- 面向初学者，需要把“为什么先查资料、为什么先电源后信号、为什么先单测后联调”讲清楚，而不是只列专业名词。
- 文档应提供可复用模板，方便后续新增任何模块时直接套用。

## 遇到的问题

- `00-硬件连接设计方法.md` 文件存在但为空，需要完整新建。
- 用户需要的是方法论而不是具体项目方案，需要避免重复 `01-总体思路.md` 中的具体接线内容。

## 解决方法

- 采用“设计思想 -> 查资料 -> 标准流程 -> 各接口要点 -> 电源地 -> 引脚选择 -> 上电检查 -> 分阶段调试 -> 文档模板”的结构。
- 提供模块信息表、引脚表、软件配置表、验证方法表等可复用模板。
- 明确最后的通用流程：查资料 -> 定电源 -> 定电平 -> 定协议 -> 选引脚 -> 画表 -> 接线 -> 单测 -> 联调 -> 记录。

# 2 安装常用软件

安装常用软件VScode

注意安装ARM64处理器版本（官网默认给出AMD64的）

安装后发现终端等功能无法正常使用，通过命令行发现提示缺少glibc 2.28以上版本

根据资料：(https://code.visualstudio.com/docs/supporting/faq),安装了1.85版本

```
VS Code 官方 FAQ 说明从 1.86.1 起 Linux 桌面版要求 glibc 2.28+，旧系统建议使用 1.85 并关闭更新
```

为了避免 VS Code 自动检查并升级到不兼容的新版本，需要关闭更新

在位置`/home/gwb/.config/Code/User/settings.json`加入：

```
  "update.mode": "none"
```

也可以在 VS Code 里手动查看：

  1. 打开 VS Code
  2. 按 Ctrl + ,
  3. 搜索 update mode
  4. 设置为 none

这是 glibc 版本不兼容。系统 glibc 版本太旧，Node 26 需要 GLIBC_2.28

`nvm install node` 默认安装 Node v26.2.0，但 Node 18+ 官方 ARM64 构建要求 GLIBC ≥ 2.28，运行时报错：
node: /lib/aarch64-linux-gnu/libc.so.6: version `GLIBC_2.28' not found (required by node)
plain

尝试下载已经编译好的源码，但是 unofficial-builds.nodejs.org 没有提供 ARM64 架构的 glibc-217 兼容构建（仅 x64），下载显示404

所以尝试源码编译 Node.js 20，在 glibc 2.27 的 ARM64 系统上，从源码编译可以生成完全兼容的二进制：

但从源码编译 Node 20 时，默认编译器 g++ 7.5.0 版本不足，需要 g++ 10.1+，因此安装了安装 gcc-10/g++-10

```
# 1. 添加 toolchain PPA
sudo apt-get install -y software-properties-common
sudo add-apt-repository -y ppa:ubuntu-toolchain-r/test
sudo apt-get update

# 2. 安装 gcc-10
sudo apt-get install -y gcc-10 g++-10
```

安装后能够正常编译，编译过程如下

```
# 1. 安装编译依赖
sudo apt-get update
sudo apt-get install -y python3 g++ make python3-pip

# 2. 下载 Node.js 20 源码
cd ~
wget https://nodejs.org/dist/v20.14.0/node-v20.14.0.tar.gz
tar -xzf node-v20.14.0.tar.gz
cd node-v20.14.0

# 3. 配置并编译（ARM64 上约需 30-90 分钟）
./configure --prefix=$HOME/.local/node
make -j$(nproc)
make install

# 4. 加 PATH
echo 'export PATH="$HOME/.local/node/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc

# 5. 验证
node -v
npm -v
```

等待1.5小时后因为实在太慢了，所以放弃

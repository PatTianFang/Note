### CUDA Toolkit

如果只需要补装 CUDA Toolkit 10.2：

```bash
sudo apt update
sudo apt install cuda-toolkit-10-2 cuda-samples-10-2
```

常用开发包：

```bash
sudo apt install cuda-compiler-10-2 cuda-nvcc-10-2 cuda-libraries-dev-10-2 cuda-cudart-dev-10-2
```

配置环境变量：

```bash
echo 'export PATH=/usr/local/cuda/bin:$PATH' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=/usr/local/cuda/lib64:$LD_LIBRARY_PATH' >> ~/.bashrc
source ~/.bashrc
```

验证：

```bash
nvcc --version
```

### OpenCV

安装 JetPack 提供的 OpenCV：

```bash
sudo apt update
sudo apt install libopencv libopencv-dev libopencv-python libopencv-samples opencv-licenses
```

验证 Python OpenCV：

```bash
python3 - <<'PY'
import cv2
print(cv2.__version__)
print(cv2.cuda.getCudaEnabledDeviceCount())
PY
```

如果 `cv2.cuda.getCudaEnabledDeviceCount()` 返回 `0`，说明当前 Python OpenCV 没有可用 CUDA 支持，或没有以 CUDA 模块方式构建。需要 `cv2.cuda` 时，应从源码重新编译 OpenCV，并启用 `WITH_CUDA=ON`、`CUDA_ARCH_BIN=7.2`、`OPENCV_DNN_CUDA=ON` 等选项。

参考：

- OpenCV Tegra CUDA 构建文档：`https://docs.opencv.org/4.x/d6/d15/tutorial_building_tegra_cuda.html`
- NVIDIA OpenCV with CUDA Jetson 项目页：`https://developer.nvidia.com/embedded/community/jetson-projects/opencv`

### CuPy

CuPy 官方预编译 wheel 主要面向 x86_64 Linux 和 Windows。Jetson/aarch64 上通常需要从源码构建，且版本要兼容 Python 3.6 和 CUDA 10.2。

建议先安装依赖：

```bash
sudo apt update
sudo apt install build-essential python3-dev python3-pip
python3 -m pip install --upgrade 'pip<22' setuptools wheel
python3 -m pip install 'numpy<1.20'
```

从源码构建安装：

```bash
export CUDA_PATH=/usr/local/cuda
export LD_LIBRARY_PATH=/usr/local/cuda/lib64:$LD_LIBRARY_PATH
python3 -m pip install --no-cache-dir --no-binary cupy cupy
```

如果使用旧版 CuPy，需要按 CuPy 文档选择仍支持 Python 3.6/CUDA 10.2 的版本。例如：

```bash
python3 -m pip install --no-cache-dir --no-binary cupy 'cupy<10'
```

验证：

```bash
python3 - <<'PY'
import cupy as cp
print(cp.__version__)
print(cp.cuda.runtime.getDeviceCount())
x = cp.arange(10)
print(x * 2)
PY
```

参考：

- CuPy 安装文档：`https://docs.cupy.dev/en/v9.6.0/install.html`

### PyCUDA

PyCUDA 需要本机 CUDA Toolkit、Python 开发头文件和编译器。当前设备已经具备基础条件，但尚未安装 PyCUDA。

安装依赖：

```bash
sudo apt update
sudo apt install build-essential python3-dev python3-pip
python3 -m pip install --upgrade 'pip<22' setuptools wheel numpy
```

安装：

```bash
export CUDA_ROOT=/usr/local/cuda
export CUDA_PATH=/usr/local/cuda
export PATH=/usr/local/cuda/bin:$PATH
export LD_LIBRARY_PATH=/usr/local/cuda/lib64:$LD_LIBRARY_PATH
python3 -m pip install --no-cache-dir pycuda
```

如果构建失败，通常需要检查：

- `nvcc` 是否在 `PATH` 中。
- `/usr/local/cuda/include/cuda.h` 是否存在。
- Python 头文件是否安装。
- `gcc/g++` 版本是否与 CUDA 10.2 兼容。

验证：

```bash
python3 - <<'PY'
import pycuda.driver as cuda
import pycuda.autoinit
print(cuda.Device.count())
print(cuda.Device(0).name())
PY
```

参考：

- PyCUDA PyPI 页面：`https://pypi.org/project/pycuda/`

### Docker / NVIDIA 容器运行时

如果希望使用容器隔离深度学习环境，可以安装 JetPack 对应的 NVIDIA 容器运行时：

```bash
sudo apt update
sudo apt install nvidia-container-runtime nvidia-container-toolkit nvidia-docker2
```

验证 Docker：

```bash
docker --version
dpkg -l | grep nvidia-container
```

JetPack 4.6 对应的 CUDA/TensorRT runtime 容器可以从 NVIDIA NGC 获取，使用前需要确认镜像 tag 与 L4T/JetPack 版本匹配。

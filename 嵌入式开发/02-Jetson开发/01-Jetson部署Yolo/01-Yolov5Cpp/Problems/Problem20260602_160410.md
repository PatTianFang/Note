# Problem20260602_160410

## 任务目标

为 Jetson AGX Xavier、Ubuntu 18.04、C++、YOLOv5、GPU/TensorRT 识别程序制定由浅入深的中文方案和技术路线，并在项目中生成多份按阅读顺序编号的 Markdown 文档。

## 思考过程

用户说明“什么也不懂”，所以文档不能直接从代码和命令开始，而应先解释 Jetson、JetPack、GPU、YOLOv5、ONNX、TensorRT engine 的关系。

关键假设：

1. 目标硬件是 NVIDIA Jetson AGX Xavier。
2. 系统是 Ubuntu 18.04，对应 JetPack 4.x 软件栈。
3. 用户希望用 C++ 做部署程序，而不是 Python 演示程序。
4. GPU 调用应优先通过 TensorRT，而不是 C++ 直接加载 PyTorch `.pt`。

主要选择：

1. 推荐“电脑或云端训练，Jetson 本机生成 TensorRT engine，C++ 加载 engine 推理”的路线。
2. 不推荐新手在 Jetson 上训练 YOLOv5，因为 Ubuntu 18.04 和现代 Python/PyTorch 依赖容易冲突。
3. 第一版采用 YOLOv5s、640 输入、batch=1、FP16，暂不做 INT8、多线程和 DeepStream。
4. 文档分为概念、环境、训练、转换、C++ 工程、摄像头、优化、实施清单，便于按顺序阅读。

## 遇到的问题

1. Jetson 教程经常混用不同 JetPack、CUDA、TensorRT 版本，容易误导用户。
2. TensorRT engine 和硬件及软件版本绑定，如果不强调本机生成，后续部署容易失败。
3. YOLOv5 的输出格式和导出参数有关，文档需要提示以实际 ONNX 输出 shape 为准。
4. 用户要求中文命名，因此文件名使用中文并按阅读顺序编号。

## 解决方法

1. 以 NVIDIA 和 Ultralytics 官方资料作为技术依据。
2. 在文档中明确 JetPack 4.6.x、CUDA 10.2、TensorRT 8.2.x 这条 Ubuntu 18.04 常见基线，并要求以板子实际 `dpkg` 查询结果为准。
3. 将路线拆成多个阶段，每阶段给出验收标准。
4. 对新手高风险路线进行明确说明，例如直接 C++ 加载 `.pt`、在 Jetson 上训练、第一版做 INT8。
5. 每份文档都围绕一个主题，降低单篇阅读难度。

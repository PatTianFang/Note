本文是整套文档的入口。你的目标可以翻译成一句工程话：

> 在 Jetson AGX Xavier 上，用 C++ 程序读取图片、视频或摄像头画面，把 YOLOv5 模型交给 GPU/TensorRT 推理，然后把识别框画出来或输出结果。

## 最推荐的技术路线

对新手最稳的路线不是在 Jetson 上直接写 Python 训练，也不是 C++ 直接加载 `.pt` 文件，而是下面这条：

```text
准备数据集
  -> 在电脑或云服务器上用 Python/PyTorch 训练 YOLOv5
  -> 导出 ONNX 文件
  -> 把 ONNX 拷贝到 Jetson AGX Xavier
  -> 在 Jetson 本机用 TensorRT 生成 .engine 文件
  -> C++ 程序加载 .engine
  -> OpenCV/GStreamer 读取画面
  -> CUDA/TensorRT 使用 GPU 推理
  -> C++ 后处理、画框、输出结果
```

这条路线的核心原因：

1. YOLOv5 的训练依赖 Python、PyTorch，放在电脑或云端更省事。
2. Jetson AGX Xavier 的 Ubuntu 18.04 通常对应 JetPack 4.x，系统环境偏老，强行在板子上装最新版训练环境容易踩坑。
3. TensorRT engine 和 GPU 架构、CUDA、TensorRT 版本绑定，最好在目标 Jetson 本机生成。
4. C++ 负责部署和实时识别，稳定、速度快、方便接工业摄像头或业务系统。

## 不推荐的新手路线

### 不推荐 1：C++ 直接读取 YOLOv5 的 `.pt`

`.pt` 是 PyTorch 权重文件，适合 Python 训练和测试。C++ 直接加载 `.pt` 需要 LibTorch，Jetson 上版本、编译、依赖都比较麻烦，速度也通常不如 TensorRT。

### 不推荐 2：在 Jetson 上从零训练

Jetson AGX Xavier 可以跑推理，也能做少量训练实验，但不适合新手把它当训练服务器。训练会占用大量显存、时间和依赖环境。更实际的做法是：电脑或云端训练，Jetson 只负责部署。

### 不推荐 3：先追求 INT8、DeepStream、多线程流水线

这些都能提升性能或工程能力，但对新手来说会增加太多变量。第一阶段只做：`YOLOv5s + FP16 + batch=1 + 640 输入 + C++ TensorRT`。

## 第一阶段的最低成功标准

不要一开始就接真实摄像头和复杂业务。先完成下面 5 件事：

1. Jetson 能正常启动，确认 JetPack、CUDA、TensorRT 版本。
2. 能运行 `trtexec`，说明 TensorRT 工具可用。
3. 有一个 YOLOv5 ONNX 文件，可以在 Jetson 上转成 `.engine`。
4. C++ 程序能读取一张图片，输出检测框。
5. C++ 程序能读取摄像头或视频，实时显示检测结果。

只要这 5 件事完成，后面才谈 FPS 优化、报警逻辑、保存图片、接数据库、接上位机。


## 版本原则

Jetson AGX Xavier + Ubuntu 18.04 通常属于 JetPack 4.x 时代。不要随便把网上 Ubuntu 20.04、Ubuntu 22.04、CUDA 11、CUDA 12、最新版 TensorRT 的教程照搬过来。

建议原则：

1. 以你板子实际安装的 JetPack 为准。
2. Ubuntu 18.04 上优先使用 JetPack 4.6.x。
3. 不要随意升级系统大版本。
4. 不要把 PC 上生成的 TensorRT engine 直接拿到 Jetson 用。
5. 不要混用多个 TensorRT、CUDA、OpenCV 版本。

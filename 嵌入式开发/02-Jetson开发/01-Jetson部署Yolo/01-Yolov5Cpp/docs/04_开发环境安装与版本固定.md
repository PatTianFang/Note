# 04 开发环境安装与版本固定

这一篇讲 Jetson 上的 C++ 开发环境。目标不是装越新越好，而是让版本稳定、能编译、能调用 TensorRT 和 OpenCV。

## 总原则

1. 优先使用 JetPack 自带 CUDA、TensorRT、OpenCV。
2. 不随意从网上下载 PC 版 CUDA。
3. 不随意使用 pip 安装 TensorRT 覆盖系统包。
4. 所有版本先查询，再安装。
5. 第一版只做 C++ TensorRT 推理，不在 Jetson 上训练。

## 更新软件源

先做普通更新：

```bash
sudo apt update
```

谨慎执行大规模升级。如果你是生产设备或工业主机，升级前应先备份系统镜像。

## 安装基础工具

```bash
sudo apt install -y \
  build-essential \
  cmake \
  git \
  pkg-config \
  gdb \
  v4l-utils
```

这些工具的用途：

| 工具 | 用途 |
| --- | --- |
| build-essential | C++ 编译器和基础编译工具 |
| cmake | 管理 C++ 工程构建 |
| git | 拉取代码 |
| pkg-config | 查询 OpenCV 等库的编译参数 |
| gdb | 调试 C++ 程序 |
| v4l-utils | 检查 USB 摄像头 |

## 安装 OpenCV 开发包

如果系统没有 OpenCV 开发头文件：

```bash
sudo apt install -y libopencv-dev
```

确认版本：

```bash
pkg-config --modversion opencv4 || pkg-config --modversion opencv
```

确认编译参数：

```bash
pkg-config --cflags --libs opencv4 || pkg-config --cflags --libs opencv
```

如果这两个命令可用，CMake 通常就能找到 OpenCV。

## 安装 GStreamer 开发包

摄像头和视频流经常需要 GStreamer：

```bash
sudo apt install -y \
  libgstreamer1.0-dev \
  libgstreamer-plugins-base1.0-dev \
  gstreamer1.0-tools \
  gstreamer1.0-plugins-base \
  gstreamer1.0-plugins-good \
  gstreamer1.0-plugins-bad
```

Jetson 上硬件加速摄像头和视频解码通常会用到 NVIDIA 的 GStreamer 插件，例如 `nvarguscamerasrc`、`nvv4l2decoder`、`nvvidconv`。

## 检查 TensorRT

查询 TensorRT 包：

```bash
dpkg -l | grep -E "nvinfer|tensorrt"
```

检查头文件：

```bash
ls /usr/include/NvInfer.h
ls /usr/include/aarch64-linux-gnu/NvInfer.h
```

两个命令有一个能找到即可。不同 JetPack 安装路径可能不同。

检查库文件：

```bash
ls /usr/lib/aarch64-linux-gnu/libnvinfer*
ls /usr/lib/aarch64-linux-gnu/libnvonnxparser*
```

检查 `trtexec`：

```bash
which trtexec
find /usr -name trtexec 2>/dev/null
```

常见位置包括：

```text
/usr/src/tensorrt/bin/trtexec
```

如果找到了，可以运行：

```bash
/usr/src/tensorrt/bin/trtexec --help
```

## 如果 JetPack 组件不完整

如果你发现 CUDA、TensorRT、cuDNN 缺失，可以先检查：

```bash
dpkg -l | grep nvidia-jetpack
```

在 JetPack 4.6 文档中，NVIDIA 提供了通过 apt 安装 JetPack 组件的方式。常见命令是：

```bash
sudo apt install -y nvidia-jetpack
```

但要注意：如果你的系统源配置不对，这条命令可能找不到包。此时应先检查 NVIDIA apt 源是否正常，而不是随便下载第三方包。

## 建议的 CMake 基础写法

后续 C++ 工程可以先用下面这种结构：

```cmake
cmake_minimum_required(VERSION 3.10)
project(yolo_jetson_cpp)

set(CMAKE_CXX_STANDARD 14)
set(CMAKE_CXX_STANDARD_REQUIRED ON)
set(CMAKE_BUILD_TYPE Release)

find_package(OpenCV REQUIRED)
find_package(CUDA REQUIRED)

include_directories(
  ${OpenCV_INCLUDE_DIRS}
  /usr/include
  /usr/include/aarch64-linux-gnu
)

link_directories(
  /usr/lib/aarch64-linux-gnu
)

add_executable(yolo_detect
  src/main.cpp
)

target_link_libraries(yolo_detect
  ${OpenCV_LIBS}
  nvinfer
  nvonnxparser
  cudart
)
```

第一版先让它能编译。后面再拆成多个 `.cpp` 和 `.h`。

## 编译测试

创建最小测试文件：

```cpp
#include <iostream>
#include <opencv2/opencv.hpp>
#include "NvInfer.h"

int main() {
    std::cout << "OpenCV: " << CV_VERSION << std::endl;
    std::cout << "TensorRT header loaded" << std::endl;
    return 0;
}
```

编译：

```bash
mkdir -p build
cd build
cmake ..
make -j$(nproc)
./yolo_detect
```

如果能输出 OpenCV 版本，并且没有链接错误，说明基础环境已可用。

## 常见环境错误

### 找不到 `NvInfer.h`

可能原因：

1. TensorRT 开发包没安装。
2. 头文件路径不在 CMake include 路径中。
3. JetPack 安装不完整。

处理：

```bash
find /usr -name NvInfer.h 2>/dev/null
```

找到后把对应路径加入 `include_directories`。

### 找不到 `libnvinfer.so`

可能原因：

1. TensorRT runtime 库缺失。
2. 链接路径不对。

处理：

```bash
find /usr -name "libnvinfer.so*" 2>/dev/null
```

通常库在：

```text
/usr/lib/aarch64-linux-gnu/
```

### OpenCV 能编译但打不开摄像头

这通常不是 OpenCV 编译问题，而是摄像头路径或 GStreamer pipeline 问题。先用 `v4l2-ctl` 或 `gst-launch-1.0` 测试摄像头。

## 本篇验收标准

完成本篇后，你应该做到：

1. `cmake` 和 `make` 能正常使用。
2. C++ 能包含 OpenCV 头文件。
3. C++ 能包含 TensorRT 头文件。
4. 程序能链接 `nvinfer`、`nvonnxparser`、`cudart`。
5. 能找到并运行 `trtexec`。

如果这些还没做到，先不要写 YOLOv5 后处理代码。

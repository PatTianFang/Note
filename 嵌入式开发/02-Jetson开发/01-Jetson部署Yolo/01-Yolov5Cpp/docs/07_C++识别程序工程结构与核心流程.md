# 07 C++ 识别程序工程结构与核心流程

这一篇讲 C++ 程序应该怎么写。目标是结构清楚、先跑通，不追求一开始就写成大型框架。

## 第一版程序只做 3 件事

第一版不要加太多功能。只做：

1. 加载 TensorRT engine。
2. 读取一张图片并推理。
3. 把检测框画到图片上。

图片推理跑通后，再扩展到视频和摄像头。

## 推荐工程结构

```text
YoloJetsonCpp/
  CMakeLists.txt
  models/
    best_fp16_640.engine
    labels.txt
  src/
    main.cpp
    TensorRTEngine.cpp
    YoloDetector.cpp

  include/
    TensorRTEngine.h
    YoloDetector.h
    Detection.h
    Preprocess.h
    Postprocess.h
  test_images/

  output/
```

各文件职责：

| 文件 | 职责 |
| --- | --- |
| `main.cpp` | 解析参数，调用检测流程 |
| `TensorRTEngine` | 加载 engine，管理 TensorRT runtime/context/buffers |
| `YoloDetector` | 把预处理、推理、后处理串起来 |
| `Preprocess` | resize、补边、BGR 转 RGB、归一化、NCHW |
| `Postprocess` | 置信度过滤、NMS、坐标映射 |
| `Detection` | 保存检测结果的数据结构 |

## 数据结构

检测结果可以先定义成这样：

```cpp
struct Detection {
    int class_id;
    float confidence;
    float x1;
    float y1;
    float x2;
    float y2;
};
```

坐标使用原图坐标，不使用 640x640 输入图坐标。这样画框和业务逻辑都方便。

## 主程序流程

```text
读取命令行参数
  -> 加载 labels.txt
  -> 加载 TensorRT engine
  -> 读取图片
  -> detector.detect(image)
  -> 遍历 Detection 结果
  -> 画框
  -> 保存 output/result.jpg
```

## 预处理流程

YOLOv5 常见输入是：

```text
1 x 3 x 640 x 640
```

OpenCV 读取图片后通常是：

```text
H x W x 3，BGR，uint8，0 到 255
```

需要变成：

```text
1 x 3 x 640 x 640，RGB，float，0.0 到 1.0
```

步骤：

1. 记录原图宽高。
2. 按比例缩放到 640 内。
3. 四周补边，也叫 letterbox。
4. BGR 转 RGB。
5. `uint8` 转 `float`。
6. 每个像素除以 255。
7. HWC 转 NCHW。

## letterbox 为什么重要

如果原图是 1280x720，直接强行 resize 成 640x640，会把画面拉变形。模型可能还能识别，但框的位置和比例会不稳定。

letterbox 的做法：

```text
原图 1280x720
  -> 等比例缩放到 640x360
  -> 上下各补 140 像素
  -> 得到 640x640
```

坐标映射回原图时必须反向处理：

```text
orig_x = (model_x - pad_x) / scale
orig_y = (model_y - pad_y) / scale
```

如果框整体上移、下移、变宽、变窄，优先检查 letterbox 反算。

## TensorRT 推理封装

`TensorRTEngine` 建议负责这些事情：

1. 从文件读取 engine。
2. 创建 `IRuntime`。
3. 反序列化 `ICudaEngine`。
4. 创建 `IExecutionContext`。
5. 查询输入输出 binding。
6. 分配 GPU 内存。
7. 创建 CUDA stream。
8. 执行推理。
9. 析构时释放资源。

伪代码：

```cpp
class TensorRTEngine {
public:
    bool load(const std::string& engine_path);
    bool infer(const float* input, float* output);

private:
    nvinfer1::IRuntime* runtime_ = nullptr;
    nvinfer1::ICudaEngine* engine_ = nullptr;
    nvinfer1::IExecutionContext* context_ = nullptr;
    cudaStream_t stream_ = nullptr;
    void* bindings_[2] = {nullptr, nullptr};
    size_t input_bytes_ = 0;
    size_t output_bytes_ = 0;
};
```

第一版可以先假设 1 个输入、1 个输出。等跑通后，再支持多个输出或动态 shape。

## 后处理流程

假设输出是：

```text
1 x 25200 x (5 + 类别数)
```

每个候选框处理：

1. 读取 `cx, cy, w, h`。
2. 读取 `objectness`。
3. 找到最大类别置信度和类别编号。
4. 计算 `score = objectness * class_confidence`。
5. 小于阈值就丢弃。
6. 把 `cx, cy, w, h` 转成 `x1, y1, x2, y2`。
7. 把 640 输入坐标映射回原图坐标。
8. 按类别做 NMS。

OpenCV 有现成 NMS：

```cpp
cv::dnn::NMSBoxes(boxes, scores, confThreshold, nmsThreshold, indices);
```

虽然我们不使用 OpenCV DNN 推理，但可以使用它的 NMS 工具。

## C++ 里最容易写错的地方

### 输入内存顺序

模型要的是 NCHW：

```text
channel first
R 全部像素
G 全部像素
B 全部像素
```

OpenCV Mat 是 HWC：

```text
pixel by pixel
BGRBGRBGR...
```

不要直接把 OpenCV Mat 的内存拷进 TensorRT 输入。

### 颜色顺序

OpenCV 默认是 BGR，YOLOv5 常用 RGB。必须转换：

```cpp
cv::cvtColor(image, rgb, cv::COLOR_BGR2RGB);
```

### 输出大小

不要硬猜输出长度。第一版可以写死，但写死前必须用 `trtexec --verbose` 或 C++ 查询 binding shape 确认。

### 坐标范围

后处理后要 clamp：

```text
x1 >= 0
y1 >= 0
x2 <= image_width - 1
y2 <= image_height - 1
```

## 从图片到视频的扩展

图片版本：

```text
读取一张图
  -> detect
  -> 保存结果
```

视频版本：

```text
打开 VideoCapture
  -> while 循环读取 frame
  -> detect
  -> draw
  -> imshow 或写文件
```

摄像头版本本质上也是视频版本，只是输入源换成 `/dev/video0` 或 GStreamer pipeline。

1. 成功加载 `.engine`。

4. 在图片上画框。

# 10 从零到上线的实施清单

这一篇是执行清单。你可以按顺序打勾，不要跳着做。

## 阶段 0：确认目标

先写清楚：

```text
我要识别什么：
类别数量：
使用图片、USB 摄像头、CSI 摄像头还是 RTSP：
期望 FPS：
是否需要保存报警图片：
是否需要输出 JSON、串口、HTTP 或数据库：
```

最低目标示例：

```text
识别安全帽、未戴安全帽、人，共 3 类。
输入是 USB 摄像头。
目标 15 FPS 以上。
识别后在画面上画框，并在终端打印类别和置信度。
```

## 阶段 1：Jetson 系统确认

执行：

```bash
cat /etc/nv_tegra_release
dpkg -l | grep nvidia-jetpack
nvcc --version
dpkg -l | grep -E "nvinfer|tensorrt"
pkg-config --modversion opencv4 || pkg-config --modversion opencv
tegrastats
```

记录：

```text
JetPack：
L4T：
CUDA：
TensorRT：
OpenCV：
```

通过标准：

1. 能进入系统。
2. 能查询 JetPack。
3. 能找到 TensorRT。
4. 能找到 OpenCV。
5. `tegrastats` 正常运行。

## 阶段 2：C++ 编译环境确认

安装：

```bash
sudo apt update
sudo apt install -y build-essential cmake git pkg-config gdb v4l-utils
sudo apt install -y libopencv-dev
```

建立最小 C++ 工程，确认：

1. 能 include OpenCV。
2. 能 include TensorRT。
3. 能链接 `nvinfer`。
4. 能编译出可执行文件。

通过标准：

```text
./yolo_detect 能运行并输出 OpenCV 版本
```

## 阶段 3：准备模型

在电脑或云端：

```bash
git clone https://github.com/ultralytics/yolov5.git
cd yolov5
pip install -r requirements.txt
```

训练：

```bash
python train.py \
  --img 640 \
  --batch 16 \
  --epochs 100 \
  --data /path/to/data.yaml \
  --weights yolov5s.pt \
  --name my_yolov5s
```

导出：

```bash
python export.py \
  --weights runs/train/my_yolov5s/weights/best.pt \
  --img 640 640 \
  --batch 1 \
  --include onnx \
  --opset 12 \
  --simplify
```

通过标准：

1. 有 `best.pt`。
2. 有 `best.onnx`。
3. 有 `labels.txt`。
4. 知道输入 shape。
5. 知道输出 shape。

## 阶段 4：Jetson 上生成 TensorRT engine

拷贝：

```bash
mkdir -p ~/models
scp best.onnx nvidia@<jetson_ip>:/home/nvidia/models/
scp labels.txt nvidia@<jetson_ip>:/home/nvidia/models/
```

生成：

```bash
cd ~/models
/usr/src/tensorrt/bin/trtexec \
  --onnx=best.onnx \
  --saveEngine=best_fp16_640.engine \
  --fp16 \
  --workspace=2048 \
  --verbose
```

测试：

```bash
/usr/src/tensorrt/bin/trtexec --loadEngine=best_fp16_640.engine
```

通过标准：

1. engine 生成成功。
2. engine 能加载。
3. `trtexec` 能输出推理耗时。

## 阶段 5：C++ 图片识别

实现：

1. 读取 `best_fp16_640.engine`。
2. 读取 `labels.txt`。
3. 读取 `demo.jpg`。
4. 做 letterbox 预处理。
5. TensorRT 推理。
6. YOLOv5 后处理。
7. 保存 `output/result.jpg`。

通过标准：

1. 终端打印检测结果。
2. `result.jpg` 中框的位置正确。
3. 类别名称正确。
4. 置信度合理。

## 阶段 6：C++ 视频识别

实现：

```bash
./yolo_detect --engine models/best_fp16_640.engine --video demo.mp4
```

通过标准：

1. 视频逐帧检测。
2. 能正常退出。
3. FPS 可打印。
4. 程序不崩溃。

## 阶段 7：摄像头接入

USB 摄像头：

```bash
./yolo_detect --engine models/best_fp16_640.engine --camera usb:0
```

CSI 摄像头：

```bash
./yolo_detect --engine models/best_fp16_640.engine --camera csi
```

RTSP：

```bash
./yolo_detect --engine models/best_fp16_640.engine --camera rtsp://user:pass@ip/stream
```

通过标准：

1. 摄像头连续运行。
2. 检测框跟随画面。
3. 延迟可接受。
4. 运行 30 分钟稳定。

## 阶段 8：性能优化

先记录分段耗时：

```text
read=
preprocess=
infer=
postprocess=
display=
total=
fps=
```

优化顺序：

1. 确认 FP16。
2. 确认电源模式。
3. 关闭显示测 FPS。
4. 降低输入尺寸。
5. 换 YOLOv5n 或 YOLOv5s。
6. 优化预处理。
7. 再考虑多线程。
8. 最后考虑 INT8。

通过标准：

```text
稳定 FPS 达到业务要求
温度和内存稳定
长时间运行不崩溃
```

## 阶段 9：部署成可长期运行程序

部署时要考虑：

1. 程序崩溃后是否自动重启。
2. 摄像头断流后是否自动重连。
3. 日志保存在哪里。
4. 磁盘满了怎么办。
5. 模型文件和配置文件是否固定。
6. 开机是否自动启动。

可以用 systemd 管理：

```ini
[Unit]
Description=YOLOv5 TensorRT Detector
After=network.target

[Service]
ExecStart=/home/nvidia/YoloJetsonCpp/build/yolo_detect --config /home/nvidia/YoloJetsonCpp/config.yaml
WorkingDirectory=/home/nvidia/YoloJetsonCpp
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

第一版调试时不要急着做开机自启。等手动运行稳定后再加。

## 最终交付物

一个可交付版本至少应包含：

1. C++ 源码。
2. `CMakeLists.txt`。
3. `.engine` 模型文件。
4. `labels.txt`。
5. 运行说明。
6. 环境版本记录。
7. 测试图片和测试结果。
8. FPS 测试结果。
9. 常见问题说明。

## 最终验收标准

建议用下面标准验收：

| 项目 | 标准 |
| --- | --- |
| 正确性 | 类别正确，框位置正确 |
| 稳定性 | 连续运行 2 到 24 小时不崩溃 |
| 性能 | 达到目标 FPS |
| 资源 | 温度、内存、功耗可接受 |
| 可维护 | 能替换模型，能看日志，能重启 |
| 可复现 | 新 Jetson 按文档能重新部署 |

## 你下一步应该做什么

如果你现在什么也没有，按这个最小顺序做：

1. 确认 JetPack、CUDA、TensorRT。
2. 安装 C++ 编译环境。
3. 用 `trtexec --help` 确认 TensorRT 工具可用。
4. 先使用公开的 YOLOv5s 模型导出 ONNX。
5. 在 Jetson 上生成 FP16 engine。
6. 写 C++ 图片识别。
7. 再接摄像头。

# 11 常用命令速查表

这一篇把常用命令集中放在一起。真正上板调试时，你可以优先查这一篇。

## Jetson 系统版本检查

```bash
cat /etc/nv_tegra_release
uname -a
lsb_release -a
dpkg -l | grep nvidia-jetpack
```

记录重点：

```text
L4T 版本
Ubuntu 版本
JetPack 版本
内核版本
```

## CUDA 检查

```bash
nvcc --version
ls /usr/local/cuda
ls /usr/local/cuda/bin/nvcc
ls /usr/local/cuda/lib64
```

如果 `nvcc` 找不到，临时添加：

```bash
export PATH=/usr/local/cuda/bin:$PATH
export LD_LIBRARY_PATH=/usr/local/cuda/lib64:$LD_LIBRARY_PATH
```

写入 `.bashrc`：

```bash
echo 'export PATH=/usr/local/cuda/bin:$PATH' >> ~/.bashrc
echo 'export LD_LIBRARY_PATH=/usr/local/cuda/lib64:$LD_LIBRARY_PATH' >> ~/.bashrc
source ~/.bashrc
```

## TensorRT 检查

```bash
dpkg -l | grep -E "nvinfer|tensorrt|nvonnxparser"
find /usr -name NvInfer.h 2>/dev/null
find /usr -name "libnvinfer.so*" 2>/dev/null
find /usr -name trtexec 2>/dev/null
```

常见 `trtexec` 路径：

```text
/usr/src/tensorrt/bin/trtexec
```

检查帮助：

```bash
/usr/src/tensorrt/bin/trtexec --help
```

## OpenCV 检查

```bash
pkg-config --modversion opencv4 || pkg-config --modversion opencv
pkg-config --cflags --libs opencv4 || pkg-config --cflags --libs opencv
```

如果缺少开发包：

```bash
sudo apt update
sudo apt install -y libopencv-dev
```

## GStreamer 检查

```bash
gst-launch-1.0 --version
gst-inspect-1.0 nvarguscamerasrc
gst-inspect-1.0 nvv4l2decoder
gst-inspect-1.0 nvvidconv
```

安装常用包：

```bash
sudo apt install -y \
  gstreamer1.0-tools \
  gstreamer1.0-plugins-base \
  gstreamer1.0-plugins-good \
  gstreamer1.0-plugins-bad \
  libgstreamer1.0-dev \
  libgstreamer-plugins-base1.0-dev
```

## C++ 编译环境检查

```bash
g++ --version
cmake --version
make --version
```

安装：

```bash
sudo apt install -y build-essential cmake git pkg-config gdb
```

标准编译：

```bash
mkdir -p build
cd build
cmake ..
make -j$(nproc)
```

## 电源模式和性能监控

查询模式：

```bash
sudo nvpmodel -q
sudo nvpmodel -q --verbose
```

切换模式：

```bash
sudo nvpmodel -m <模式编号>
```

锁频：

```bash
sudo jetson_clocks
```

监控：

```bash
tegrastats
```

注意：模式编号不要死记，先用 `nvpmodel -q --verbose` 看你板子当前支持什么。

## USB 摄像头检查

```bash
ls /dev/video*
sudo apt install -y v4l-utils
v4l2-ctl --list-devices
v4l2-ctl -d /dev/video0 --list-formats-ext
```

OpenCV 简单测试可以写一个最小程序，或先用系统播放器、GStreamer 测。

## CSI 摄像头检查

最简单显示：

```bash
gst-launch-1.0 nvarguscamerasrc ! nvoverlaysink
```

转 BGR 给 OpenCV：

```bash
gst-launch-1.0 nvarguscamerasrc ! \
  video/x-raw\(memory:NVMM\),width=1280,height=720,framerate=30/1 ! \
  nvvidconv ! video/x-raw,format=BGRx ! \
  videoconvert ! video/x-raw,format=BGR ! \
  fakesink
```

如果这一步失败，优先排查摄像头排线、驱动、系统镜像，不要先改 C++。

## RTSP 检查

网络连通：

```bash
ping <摄像头IP>
```

RTSP 基础测试：

```bash
gst-launch-1.0 rtspsrc location=rtsp://user:pass@ip/stream latency=100 ! fakesink
```

H.264 硬件解码测试：

```bash
gst-launch-1.0 rtspsrc location=rtsp://user:pass@ip/stream latency=100 ! \
  rtph264depay ! h264parse ! nvv4l2decoder ! nvvidconv ! \
  video/x-raw,format=BGRx ! videoconvert ! video/x-raw,format=BGR ! \
  fakesink
```

## YOLOv5 训练命令

在电脑或云端执行：

```bash
python train.py \
  --img 640 \
  --batch 16 \
  --epochs 100 \
  --data /path/to/data.yaml \
  --weights yolov5s.pt \
  --name my_yolov5s
```

显存不够就减小 batch：

```bash
--batch 8
--batch 4
--batch 2
```

## YOLOv5 导出 ONNX

```bash
python export.py \
  --weights runs/train/my_yolov5s/weights/best.pt \
  --img 640 640 \
  --batch 1 \
  --include onnx \
  --opset 12 \
  --simplify
```

如果 `--simplify` 报错：

```bash
python export.py \
  --weights runs/train/my_yolov5s/weights/best.pt \
  --img 640 640 \
  --batch 1 \
  --include onnx \
  --opset 12
```

## ONNX 拷贝到 Jetson

```bash
scp best.onnx nvidia@<jetson_ip>:/home/nvidia/models/
scp labels.txt nvidia@<jetson_ip>:/home/nvidia/models/
```

## TensorRT 转 engine

```bash
cd ~/models
/usr/src/tensorrt/bin/trtexec \
  --onnx=best.onnx \
  --saveEngine=best_fp16_640.engine \
  --fp16 \
  --workspace=2048 \
  --verbose
```

动态 shape 示例：

```bash
/usr/src/tensorrt/bin/trtexec \
  --onnx=best.onnx \
  --saveEngine=best_fp16_640.engine \
  --fp16 \
  --minShapes=images:1x3x640x640 \
  --optShapes=images:1x3x640x640 \
  --maxShapes=images:1x3x640x640 \
  --workspace=2048
```

## engine 测试

```bash
/usr/src/tensorrt/bin/trtexec --loadEngine=best_fp16_640.engine
```

如果加载失败，重新在当前 Jetson 上生成 engine。

## 快速排错顺序

1. 系统版本是否正确。
2. TensorRT 是否可用。
3. `trtexec` 是否能加载 engine。
4. C++ 是否能加载 engine。
5. 图片版是否正确。
6. 视频版是否正确。
7. 摄像头单独是否能打开。
8. 推理 FPS 是否正常。
9. 温度、电源、内存是否正常。

# 12 C++ 核心代码模板与接口设计

这一篇不是完整代码，而是告诉你 C++ 工程应该有哪些类、函数和数据结构。后续真正编码时，按这个结构写会更稳。

## 第一版设计目标

第一版只支持：

1. batch=1。
2. 固定输入尺寸，例如 640x640。
3. 一个 TensorRT engine 文件。
4. 一个输入 tensor。
5. 一个输出 tensor。
6. 图片、视频或摄像头单路输入。

不要第一版就支持动态 batch、多模型、多路视频、多线程和 INT8 校准。

## Detection.h

```cpp
#pragma once

#include <string>

struct Detection {
    int class_id = -1;
    float confidence = 0.0f;
    float x1 = 0.0f;
    float y1 = 0.0f;
    float x2 = 0.0f;
    float y2 = 0.0f;
};
```

说明：

1. 坐标用原图坐标。
2. `class_id` 对应 `labels.txt` 的行号。
3. `confidence` 是最终分数，通常是 `objectness * class_score`。

## Preprocess.h

```cpp
#pragma once

#include <opencv2/opencv.hpp>
#include <vector>

struct PreprocessMeta {
    float scale = 1.0f;
    float pad_x = 0.0f;
    float pad_y = 0.0f;
    int input_w = 640;
    int input_h = 640;
    int original_w = 0;
    int original_h = 0;
};

struct PreprocessOutput {
    std::vector<float> input;
    PreprocessMeta meta;
};

PreprocessOutput preprocessYoloV5(
    const cv::Mat& image,
    int input_w,
    int input_h
);
```

为什么要返回 `meta`：

1. 后处理要把 640 坐标还原到原图。
2. 必须知道缩放比例和补边大小。
3. 不保存这些信息，框很容易偏。

## Postprocess.h

```cpp
#pragma once

#include "Detection.h"
#include "Preprocess.h"
#include <vector>

std::vector<Detection> postprocessYoloV5(
    const float* output,
    int num_predictions,
    int num_classes,
    float conf_threshold,
    float nms_threshold,
    const PreprocessMeta& meta
);
```

参数解释：

| 参数 | 含义 |
| --- | --- |
| `output` | TensorRT 输出数组 |
| `num_predictions` | 候选框数量，例如 25200 |
| `num_classes` | 类别数 |
| `conf_threshold` | 置信度阈值 |
| `nms_threshold` | NMS 阈值 |
| `meta` | 预处理时保存的缩放和补边信息 |

## TensorRTEngine.h

```cpp
#pragma once

#include <string>
#include <vector>

class TensorRTEngine {
public:
    TensorRTEngine() = default;
    ~TensorRTEngine();

    bool load(const std::string& engine_path);

    bool infer(
        const float* input,
        std::vector<float>& output
    );

    int inputWidth() const { return input_w_; }
    int inputHeight() const { return input_h_; }
    int outputSize() const { return output_size_; }

private:
    bool allocateBuffers();
    void release();

private:
    void* runtime_ = nullptr;
    void* engine_ = nullptr;
    void* context_ = nullptr;
    void* stream_ = nullptr;

    void* device_input_ = nullptr;
    void* device_output_ = nullptr;

    int input_w_ = 640;
    int input_h_ = 640;
    int input_size_ = 0;
    int output_size_ = 0;
};
```

上面用 `void*` 是为了表达接口结构。实际写代码时应使用 TensorRT 和 CUDA 类型，例如：

```cpp
nvinfer1::IRuntime*
nvinfer1::ICudaEngine*
nvinfer1::IExecutionContext*
cudaStream_t
```

## TensorRTEngine 的职责边界

这个类只做 TensorRT 相关事情：

1. 加载 engine。
2. 申请显存。
3. 拷贝输入。
4. 执行推理。
5. 拷贝输出。

它不应该做：

1. 读摄像头。
2. 画框。
3. 解析 YOLOv5 输出。
4. 管理业务报警逻辑。

职责分开，后续排错更容易。

## YoloDetector.h

```cpp
#pragma once

#include "Detection.h"
#include "TensorRTEngine.h"
#include <opencv2/opencv.hpp>
#include <vector>

class YoloDetector {
public:
    bool init(
        const std::string& engine_path,
        int input_w,
        int input_h,
        int num_classes,
        float conf_threshold,
        float nms_threshold
    );

    std::vector<Detection> detect(const cv::Mat& image);

private:
    TensorRTEngine engine_;
    int input_w_ = 640;
    int input_h_ = 640;
    int num_classes_ = 0;
    float conf_threshold_ = 0.25f;
    float nms_threshold_ = 0.45f;
};
```

`YoloDetector` 的职责是把三件事串起来：

```text
preprocess -> TensorRT infer -> postprocess
```

## main.cpp 流程模板

```cpp
int main(int argc, char** argv) {
    std::string engine_path = "models/best_fp16_640.engine";
    std::string labels_path = "models/labels.txt";
    std::string image_path = "test_images/demo.jpg";

    auto labels = loadLabels(labels_path);

    YoloDetector detector;
    bool ok = detector.init(
        engine_path,
        640,
        640,
        static_cast<int>(labels.size()),
        0.25f,
        0.45f
    );

    if (!ok) {
        return 1;
    }

    cv::Mat image = cv::imread(image_path);
    if (image.empty()) {
        return 1;
    }

    auto detections = detector.detect(image);
    drawDetections(image, detections, labels);
    cv::imwrite("output/result.jpg", image);
    return 0;
}
```

## CUDA 错误检查宏

建议加入：

```cpp
#define CHECK_CUDA(call)                                                   \
    do {                                                                   \
        cudaError_t err = (call);                                           \
        if (err != cudaSuccess) {                                           \
            std::cerr << "CUDA error: " << cudaGetErrorString(err)          \
                      << " at " << __FILE__ << ":" << __LINE__ << std::endl;\
            std::exit(1);                                                   \
        }                                                                  \
    } while (0)
```

没有错误检查时，显存大小错、拷贝失败、流同步失败都很难定位。

## TensorRT 日志类

TensorRT 需要 logger：

```cpp
class Logger : public nvinfer1::ILogger {
public:
    void log(Severity severity, const char* msg) noexcept override {
        if (severity <= Severity::kWARNING) {
            std::cout << "[TensorRT] " << msg << std::endl;
        }
    }
};
```

调试阶段可以打印更多日志，部署阶段减少日志。

## 不要一开始就写的功能

第一版先不要写：

1. 多模型切换。
2. 动态输入尺寸。
3. INT8 校准。
4. 多线程推理。
5. 多路摄像头。
6. Web 页面。
7. 数据库。

这些都可以后续加，但第一版目标是验证核心链路。

## 本篇验收标准

你后续写代码时，至少应满足：

1. `TensorRTEngine` 只负责 engine 和推理。
2. `Preprocess` 只负责图像转模型输入。
3. `Postprocess` 只负责输出转检测框。
4. `YoloDetector` 负责串联流程。
5. `main.cpp` 保持简短。
